Tutorial

Learn how to build a complete application with Aural UI. This tutorial will guide you through creating a user profile card with multiple components working together.

What We'll Build

In this tutorial, you'll create a user profile card that includes:

  • Card component for the container
  • Badge components for status indicators
  • Button components for actions
  • Design tokens for consistent spacing and colors

Step 1: Set Up Your HTML

First, create a basic HTML file with Aural UI included:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>User Profile Tutorial</title>

    <!-- Aural UI CSS -->
    <link rel="stylesheet" href="aural-ui.css">
    <link rel="stylesheet" href="dark.css" id="theme-link">
</head>
<body>
    <!-- We'll add content here -->
</body>
</html>
Step 2: Create the Card Container

Add a card component to hold our profile. We'll use design tokens for spacing and sizing:

<div class="card" style="max-width: 500px; margin: 2rem auto; padding: var(--space-6);">
    <!-- Content will go here -->
</div>

The card class provides the background, border, and shadow. We use:

  • max-width: 500px - Prevents the card from getting too wide
  • margin: 2rem auto - Centers the card horizontally
  • padding: var(--space-6) - Adds consistent internal spacing using design tokens
Step 3: Add the Profile Header

Create the header section with an avatar and user information:

<div style="display: flex; align-items: start; gap: var(--space-4); margin-bottom: var(--space-4);">
    <!-- Avatar -->
    <div style="width: 64px; height: 64px; border-radius: var(--radius-full);
                background: linear-gradient(135deg, var(--color-primary), var(--color-secondary));
                display: flex; align-items: center; justify-content: center;
                font-size: var(--text-2xl); font-weight: var(--font-bold);
                color: white; flex-shrink: 0;">
        JD
    </div>

    <!-- User Info -->
    <div style="flex: 1;">
        <div style="display: flex; align-items: center; gap: var(--space-2); margin-bottom: var(--space-2);">
            <h3 style="font-size: var(--text-xl); font-weight: var(--font-semibold);
                       color: var(--color-text-primary); margin: 0;">Jane Doe</h3>
            <span class="badge badge-success">Active</span>
        </div>
        <p style="color: var(--color-text-secondary); margin: 0; font-size: var(--text-sm);">
            Product Designer
        </p>
    </div>
</div>

Key design tokens used:

  • var(--space-4) - Spacing between avatar and text
  • var(--radius-full) - Makes the avatar circular
  • var(--color-primary) and var(--color-secondary) - Theme colors
  • var(--text-xl), var(--text-sm) - Consistent font sizes
  • var(--color-text-primary), var(--color-text-secondary) - Text colors
Step 4: Add the Bio Section

Add a description below the header:

<p style="color: var(--color-text-secondary); margin-bottom: var(--space-4);">
    Passionate about creating accessible and beautiful user experiences.
    5 years of experience in design systems.
</p>
Step 5: Add Action Buttons

Finally, add buttons for user actions:

<div style="display: flex; gap: var(--space-3);">
    <button class="btn btn-primary">Follow</button>
    <button class="btn btn-secondary">Message</button>

Key Takeaways

This tutorial demonstrated several important concepts:

Design Tokens
  • Use spacing tokens (var(--space-*)) for consistent spacing
  • Use color tokens (var(--color-*)) for theme-aware colors
  • Use typography tokens (var(--text-*)) for consistent text sizes
  • Use radius tokens (var(--radius-*)) for consistent border radii
Component Classes
  • card - Container with background, border, and shadow
  • badge - Small status indicators with variants (success, primary, etc.)
  • btn - Interactive buttons with variants (primary, secondary, etc.)
Responsive Design
  • Use max-width to prevent components from getting too wide
  • Use flexbox for layouts that adapt to content size
  • Design tokens automatically adapt to different screen sizes
Next Steps
