Aural UI

A modern, accessible design system built for flexibility and customization

Aural UI is a comprehensive component library that combines beautiful design with practical functionality. Built on a foundation of design tokens and CSS custom properties, it offers unparalleled customization while maintaining consistency across your entire application.

Token-Driven

Customize every aspect of the design system with CSS custom properties. Change colors, spacing, typography, and more without touching component code.

Accessibility First

Built with WCAG AA compliance, keyboard navigation, proper ARIA attributes, and screen reader support out of the box.

Lightweight

Zero dependencies, vanilla JavaScript for interactive components, and optimized CSS for fast load times and smooth performance.

20+ Components

From basic buttons and inputs to advanced components like modals, dropdowns, and accordions - everything you need to build modern interfaces.

Dark Mode Ready

Switch between light and dark themes seamlessly. All components are designed to look great in both themes.

Developer Friendly

Simple, semantic HTML with intuitive class names. Copy-paste ready code snippets with clear documentation.

Installation

Get started with Aural UI using CDN, npm, or direct download

CDN (Fastest)

The quickest way to get started. Include these links in your HTML: 

<!-- Aural UI CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/aural-ui@1.0.0/dist/aural-ui.css">

<!-- Choose a theme -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/aural-ui@1.0.0/themes/dark.css">

<!-- Aural UI JavaScript -->
<script src="https://cdn.jsdelivr.net/npm/aural-ui@1.0.0/dist/aural-ui.js"></script>

<!-- Optional: Lucide Icons -->
<script src="https://unpkg.com/lucide@latest"></script>
<script>lucide.createIcons();</script>
NPM (Recommended)

Install via npm for better version control and bundling: 

# Install package
npm install aural-ui

# Then in your JavaScript
import 'aural-ui/dist/aural-ui.css';
import 'aural-ui/themes/dark.css';
import Aural from 'aural-ui';

Responsive Grid System

Aural UI includes a powerful, mobile-first grid system for creating responsive layouts with ease. 

<!-- Container -->
<div class="container">
  <!-- Content with max-width and responsive padding -->
</div>

<!-- Basic Grid (1 column on mobile, 3 on desktop) -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-6">
  <div>Column 1</div>
  <div>Column 2</div>
  <div>Column 3</div>
</div>

<!-- Auto-responsive grid (no media queries needed) -->
<div class="grid-cards">
  <div class="card">Card 1</div>
  <div class="card">Card 2</div>
  <div class="card">Card 3</div>
</div>

<!-- 12-column grid with custom spans -->
<div class="grid grid-cols-12 gap-4">
  <div class="col-span-12 md:col-span-8">Main content (8 cols on desktop)</div>
  <div class="col-span-12 md:col-span-4">Sidebar (4 cols on desktop)</div>
</div>

<!-- Common patterns -->
<div class="grid-2-col">...</div>  <!-- 2 columns on tablet+ -->
<div class="grid-3-col">...</div>  <!-- 3 columns on desktop -->
<div class="grid-4-col">...</div>  <!-- 2 on tablet, 4 on desktop -->

Breakpoints:

  • sm: 640px (mobile landscape)
  • md: 768px (tablet)
  • lg: 1024px (desktop)
  • xl: 1280px (large desktop)

Best UX Practices:

  • Use .container for content with max-width (better readability)
  • Use .grid-cards for auto-responsive card layouts (no breakpoints needed)
  • Stack content on mobile with grid-cols-1, then expand on larger screens
  • Use gap-4 (16px) for comfortable spacing, gap-6 (24px) for more breathing room
  • For complex layouts, use the 12-column grid with custom spans

Using Icons

Aural UI is designed to work seamlessly with Lucide Icons, a free and open-source icon library (MIT License). Simply include the Lucide script and use the data-lucide attribute. 

<!-- Include Lucide Icons -->
<script src="https://unpkg.com/lucide@latest"></script>

<!-- Initialize icons when DOM is loaded -->
<script>
  document.addEventListener('DOMContentLoaded', () => {
    lucide.createIcons();
  });
</script>

<!-- Use icons in your HTML -->
<i data-lucide="heart" style="width: 20px; height: 20px;"></i>
<i data-lucide="star" style="width: 24px; height: 24px; color: var(--color-warning);"></i>

<!-- Icons in buttons -->

  <i data-lucide="download" style="width: 18px; height: 18px;"></i>

Customization with Tokens

All components are built with CSS custom properties (design tokens), making customization simple and powerful. Override tokens in your own CSS or use the live token customizer in the sidebar. 

:root {
  /* Override any token */

  --radius-md: 12px;

JavaScript API Reference

Complete API documentation for all interactive components

Aural UI includes built-in JavaScript utilities for interactive components. All methods are available on the global Aural object.

Toast API

Display temporary notification messages to users. Toasts automatically dismiss after a set duration. 

// Basic usage
Aural.showToast(message, type, duration);

// Parameters:
// - message (string): The text to display
// - type (string): 'success', 'error', 'info', or 'warning'
// - duration (number, optional): Time in ms (default: 3000)

// Examples:
Aural.showToast('Changes saved!', 'success');
Aural.showToast('Please try again', 'error');
Aural.showToast('New update available', 'info');
Aural.showToast('Low disk space', 'warning');
Aural.showToast('Quick message', 'success', 2000); // 2 seconds
Modal API

Control modal dialogs programmatically. Modals can be opened, closed, or toggled. 

// Open a modal
Aural.openModal(modalId);

// Close a modal
Aural.closeModal(modalId);

// Toggle a modal (open if closed, close if open)
Aural.toggleModal(modalId);

// Parameters:
// - modalId (string): The ID of the modal element

// Examples:
Aural.openModal('my-modal');
Aural.closeModal('my-modal');
Aural.toggleModal('settings-modal');

// Modal HTML structure:
<div class="modal-overlay" id="my-modal">
  <div class="modal">
    <div class="modal-header">
      <h3>Modal Title</h3>
      <button class="modal-close" onclick="Aural.closeModal('my-modal')">Ã—</button>
    </div>
    <div class="modal-body">Content here</div>
    <div class="modal-footer">
      <button class="btn btn-primary">Confirm</button>
    </div>
  </div>
</div>
Tabs API

Initialize and control tabbed interfaces. Switch between tabs programmatically. 

// Initialize tabs when DOM is ready
document.addEventListener('DOMContentLoaded', () => {
  Aural.initTabs();
});

// Switch to a specific tab programmatically
Aural.switchTab(tabButton, tabsContainer);

// Parameters:
// - tabButton (Element): The tab button element
// - tabsContainer (Element): The tabs container element

// HTML structure:
<div class="tabs">
  <div class="tabs-list" role="tablist">
    <button class="tab" role="tab" data-tab="tab1">Tab 1</button>
    <button class="tab" role="tab" data-tab="tab2">Tab 2</button>
  </div>
  <div class="tab-panel active" data-tab-panel="tab1">Content 1</div>
  <div class="tab-panel" data-tab-panel="tab2">Content 2</div>
</div>
Tooltips API

Initialize tooltips for elements with data-tooltip attribute. 

// Initialize all tooltips when DOM is ready
document.addEventListener('DOMContentLoaded', () => {
  Aural.initTooltips();
});

// HTML usage - just add data-tooltip attribute:
<button data-tooltip="Helpful information">Hover me</button>
<span data-tooltip="More details">Info</span>

// Position tooltips (optional):
<button data-tooltip="Above" data-tooltip-position="top">Top</button>
<button data-tooltip="Below" data-tooltip-position="bottom">Bottom</button>
<button data-tooltip="Left" data-tooltip-position="left">Left</button>
<button data-tooltip="Right" data-tooltip-position="right">Right</button>

Accordion API

Toggle accordion panels to show or hide content. 

// Toggle an accordion panel
Aural.toggleAccordion(panelId);

// Parameter:
// - panelId (string): The ID of the accordion panel to toggle

// HTML structure:

Custom Theme Creation

Create your own custom theme by overriding design tokens. Here's a complete guide. 

/* Create a custom theme in your CSS */
:root {
  /* Step 1: Define your brand colors */
  --color-primary: #ff6b6b;
  --color-secondary: #4ecdc4;

  /* Step 2: Customize background colors */
  --color-bg-primary: #1a1a2e;
  --color-bg-secondary: #16213e;
  --color-bg-tertiary: #0f3460;

  /* Step 3: Set text colors for contrast */
  --color-text-primary: #eee;
  --color-text-secondary: #aaa;
  --color-text-muted: #888;

  /* Step 4: Define border colors */
  --color-border: #2a2a3e;
  --color-border-hover: #3a3a4e;

  /* Step 5: Customize spacing (optional) */
  --space-4: 20px; /* Increase default spacing */

  /* Step 6: Adjust border radius (optional) */
  --radius-md: 12px; /* More rounded corners */

  /* Step 7: Change typography (optional) */
  --font-sans: 'Inter', sans-serif;
  --text-base: 17px; /* Slightly larger text */
}

/* Apply theme with data attribute */
[data-theme="custom"] {
  --color-primary: #ff6b6b;
  --color-secondary: #4ecdc4;
  /* ... rest of theme tokens */
}

/* Toggle themes with JavaScript */
<script>
  function setTheme(theme) {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  }

  // Load saved theme
  const savedTheme = localStorage.getItem('theme') || 'dark';
  setTheme(savedTheme);
</script>

<!-- Theme switcher -->
<button onclick="setTheme('dark')">Dark</button>
<button onclick="setTheme('light')">Light</button>
<button onclick="setTheme('custom')">Custom</button>

Theme Best Practices

  • Maintain proper contrast ratios (WCAG AA: 4.5:1 for text, AAA: 7:1 for body text)
  • Test your theme with all component variants (buttons, cards, inputs, etc.)
  • Use the Token Controller in the sidebar to preview changes in real-time
  • Store user's theme preference in localStorage for persistence
  • Consider respecting user's system preference with prefers-color-scheme
  • Provide a theme toggle for better user experience
