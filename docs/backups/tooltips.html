Best Practices

When to Use

  • Use tooltips for icon-only buttons that need labels
  • Provide additional context for form fields without cluttering the UI
  • Explain status indicators or colored elements
  • Show keyboard shortcuts for actions
  • Display full text for truncated content

When Not to Use

  • Don't use tooltips for critical information - it should be visible by default
  • Avoid tooltips on mobile where hover isn't available (use tap or visible labels)
  • Don't repeat visible text in a tooltip
  • Avoid tooltips with lots of text - consider a popover or modal instead

Accessibility

  • Tooltips appear on both hover and focus for keyboard users
  • Always provide aria-label on icon-only buttons in addition to tooltips
  • Tooltip content is added to the accessibility tree via aria-describedby
  • Use semantic color variants (success, warning, error) consistently
  • Keep tooltip text concise - aim for 1-2 short sentences maximum
  • Don't rely solely on tooltips to convey critical information

Position Guidelines

  • data-tooltip-position="top" - Default, appears above element
  • data-tooltip-position="bottom" - Use when top would go off-screen
  • data-tooltip-position="left" - Use in right-aligned toolbars
  • data-tooltip-position="right" - Use for left-aligned help icons
  • Tooltips auto-adjust if they would overflow the viewport

Color Usage

  • Default (no color) - General information and help text
  • data-tooltip-color="primary" - Highlight important features
  • data-tooltip-color="success" - Positive status or completion
  • data-tooltip-color="warning" - Cautions or degraded states
  • data-tooltip-color="error" - Errors or offline status

Mobile Considerations

  • Tooltips don't work well on touch devices - use visible labels instead
  • On mobile, show critical information directly rather than in tooltips
  • Consider using popovers with explicit tap targets for mobile
  • Icon-only buttons should have visible labels on mobile screens
