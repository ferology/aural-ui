Combobox

A searchable dropdown component that combines an input field with a filterable list of options. Perfect for large datasets where users need to search and select from many items.

Best Practices

When to Use

  • Large datasets: When you have many options (20+) that would be overwhelming in a regular select dropdown
  • Search functionality needed: When users benefit from typing to filter through options quickly
  • Autocomplete behavior: When you want to suggest completions as users type
  • Dynamic or async data: When options need to be loaded from a server or API
  • Complex option display: When options need to show additional information like descriptions
  • Custom value creation: When users should be able to enter values not in the predefined list

When NOT to Use

  • Small option lists: Use a regular <select> element for 5-10 options or fewer - it's simpler and more accessible
  • Binary choices: Use radio buttons, checkboxes, or switches for yes/no or on/off choices
  • Date selection: Use a proper date picker component instead
  • Known short lists: If users are familiar with a short list of options, a select dropdown is more efficient
  • Multi-select needed: Consider a multi-select component or checkbox list instead

API Reference

initCombobox(comboboxId, options)

Initialize a combobox component with the given ID and configuration options.

Parameters:

  • comboboxId (string) - The ID of the combobox container element
  • options (object) - Configuration options:
    • options (array) - Array of option objects with value, label, and optional description
    • searchable (boolean) - Enable search filtering (default: true)
    • creatable (boolean) - Allow creating new options (default: false)
    • placeholder (string) - Input placeholder text (default: "Search...")
    • onChange (function) - Callback when selection changes, receives selected option
    • onSearch (function) - Callback when search query changes, receives query and filtered results
    • onCreate (function) - Callback when new option is created, receives the new value

Common Patterns

Location/Address Search

Perfect for city, address, or location selection. Use descriptions to show additional location information like region and country.

Tag/Category Selection

Allow users to search existing tags or create new ones on the fly. Enable the creatable option and provide an onCreate callback.

User/Mention Autocomplete

Search and mention users in text fields. Use descriptions to show email addresses or user IDs alongside names.

Product/Inventory Search

Search through large product catalogs with rich results showing SKU, price, and availability in the description field.

Search Callbacks

Use the onSearch callback to track search queries, implement analytics, or load async results from an API.
