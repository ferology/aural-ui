Toast

Temporary notification messages that appear at the corner of the screen.

Basic Toasts
View Code 
// Show a success toast
Aural.showToast('Operation completed successfully!', 'success');

// Show an error toast
Aural.showToast('Error: Failed to save changes.', 'error');
All Types
View Code 
Aural.showToast('Success message', 'success');
Aural.showToast('Informational message', 'info');
Aural.showToast('Warning message', 'warning');
Aural.showToast('Error message', 'error');
Toast with Title
View Code 
// Third parameter is the title
Aural.showToast('Your profile has been updated.', 'success', 'Success');
Aural.showToast('Unable to connect to server.', 'error', 'Connection Error');
Custom Duration
View Code 
// Fourth parameter is duration in milliseconds
Aural.showToast('Quick message', 'info', null, 2000);  // 2 seconds
Aural.showToast('Extended message', 'info', null, 8000);  // 8 seconds
Multiple Toasts
View Code 
// Toasts stack automatically
Aural.showToast('First notification', 'success');
setTimeout(() => Aural.showToast('Second notification', 'info'), 500);
setTimeout(() => Aural.showToast('Third notification', 'warning'), 1000);

Multiple toasts automatically stack in the corner, with newer toasts appearing below existing ones.

Common Use Cases

Best Practices

When to Use

  • Brief feedback for user actions (save, delete, copy)
  • Status updates that don't require user action
  • Background process completions
  • Non-critical notifications

Toast Types

  • success - Successful operations
  • info - Informational updates
  • warning - Warnings that don't block user
  • error - Errors and failures

JavaScript API

  • Aural.showToast(message, type) - Basic toast
  • Aural.showToast(message, type, title) - With title
  • Aural.showToast(message, type, title, duration) - Custom duration

Duration Guidelines

  • Default: 4000ms (4 seconds)
  • Short messages: 2000-3000ms
  • Important messages: 6000-8000ms
  • Keep duration long enough to read

Accessibility

  • Toasts are announced to screen readers
  • Don't put critical actions in toasts
  • Ensure sufficient display time
  • Use clear, concise messages
