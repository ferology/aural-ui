Creating Custom Themes

Build your own custom theme by overriding CSS custom properties. Start with a base theme and customize the colors, spacing, and other tokens.

Step 1: Load a Base Theme

<!-- Load dark or light theme as your base -->
<link rel="stylesheet" href="./dark.css">

Step 2: Override Design Tokens

<style>
  :root {
    /* Override primary color */
    --color-primary: #8B5CF6; /* Purple */
    --color-primary-hover: #7C3AED;
    --color-primary-alpha: rgba(139, 92, 246, 0.1);

    /* Override border radius for rounder components */
    --radius-md: 1rem;
    --radius-lg: 1.5rem;

    /* Customize spacing */
    --space-4: 1.25rem; /* Increase base spacing */
  }
</style>

Pro Tip: Use the Design Tokens reference to see all available customization options.

  6. Export your theme as CSS or copy the code
  7. Add the custom CSS to your project

Dynamic Theme Switching

Allow users to switch between themes at runtime. Here's a simple implementation using JavaScript.

<!-- HTML -->
<link rel="stylesheet" href="./dark.css" id="theme-link">
<button onclick="toggleTheme()">Toggle Theme</button>

<!-- JavaScript -->
<script>
  function toggleTheme() {
    const themeLink = document.getElementById('theme-link');
    const currentTheme = themeLink.getAttribute('href');

    if (currentTheme.includes('dark')) {
      themeLink.setAttribute('href', './light.css');
      localStorage.setItem('theme', 'light');
    } else {
      themeLink.setAttribute('href', './dark.css');
      localStorage.setItem('theme', 'dark');
    }
  }

  // Load saved theme preference
  const savedTheme = localStorage.getItem('theme');
  if (savedTheme) {
    document.getElementById('theme-link')
      .setAttribute('href', `./${savedTheme}.css`);
  }
</script>

Try it now! Click the theme toggle button in the top right corner to see theme switching in action.

Best Practices

Do

  • Test themes in both light and dark modes
  • Ensure WCAG AA contrast standards (4.5:1)
  • Save user theme preferences
  • Use semantic token names
  • Document custom theme tokens

Don't

  • Use hardcoded color values
  • Override base component styles directly
  • Create themes with poor contrast
  • Forget to test on mobile devices
  • Mix multiple theming approaches
