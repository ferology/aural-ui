Theme Guide

Aural UI provides flexible theming through CSS custom properties. Choose from pre-built themes or create your own to match your brand identity.

Pre-built Themes

Aural UI includes four professionally designed themes, each optimized for different needs:

Dark Theme (Default)

Perfect for modern applications. Reduces eye strain in low-light environments with carefully balanced contrast ratios.

<link rel="stylesheet" href="dark.css">
  • Primary: Emerald green (#5ebd8f)
  • Secondary: Cyan (#06b6d4)
  • Background: Deep charcoal (#0f0f0f)
  • Contrast: WCAG AA compliant

Light Theme

Traditional and professional. Great for daytime use and print-friendly interfaces with excellent readability.

<link rel="stylesheet" href="light.css">
  • Primary: Emerald green (#5ebd8f)
  • Secondary: Cyan (#06b6d4)
  • Background: White (#ffffff)
  • Contrast: WCAG AA compliant

High Contrast Theme

Maximum accessibility for users with low vision. Pure black/white with enhanced contrast ratios and larger focus indicators.

<link rel="stylesheet" href="high-contrast.css">
  • Primary: Bright blue (#0096ff)
  • Success: Bright green (#00ff00)
  • Error: Bright red (#ff0000)
  • Background: Pure black (#000000)
  • Text: Pure white (#ffffff)
  • Contrast: WCAG AAA compliant (7:1+)
  • Focus outlines: Enhanced 3px borders

Best for: Users with low vision, visual impairments, or using the interface in bright sunlight.

Colorblind-Friendly Theme

Optimized for users with deuteranopia, protanopia, or tritanopia. Eliminates problematic red/green combinations.

<link rel="stylesheet" href="colorblind-friendly.css">
  • Success: Blue (#1a8cff) instead of green
  • Error: Orange (#ff8c1a) instead of red
  • Warning: Yellow (#ffde1a)
  • Info: Cyan (#1ad6ff)
  • Additional indicators: Border accents and icon-based indicators

Best for: Users with color vision deficiency, especially red-green colorblindness (affects ~8% of men and ~0.5% of women).

Creating Custom Themes

Build your own custom theme by overriding CSS custom properties. Start with a base theme and customize the colors, spacing, and other tokens.

Step 1: Load a Base Theme

<!-- Load dark or light theme as your base -->
<link rel="stylesheet" href="./dark.css">

Step 2: Override Design Tokens

<style>
  :root {
    /* Override primary color */
    --color-primary: #8B5CF6; /* Purple */
    --color-primary-hover: #7C3AED;
    --color-primary-alpha: rgba(139, 92, 246, 0.1);

    /* Override border radius for rounder components */
    --radius-md: 1rem;
    --radius-lg: 1.5rem;

    /* Customize spacing */
    --space-4: 1.25rem; /* Increase base spacing */
  }
</style>

Pro Tip: Use the Design Tokens reference to see all available customization options.

Visual Theme Builder

Aural UI includes a powerful visual theme builder that lets you create custom themes without writing CSS. Perfect for designers, developers, and anyone who wants to customize the look and feel.

Features

  • Visual Color Pickers: Choose colors using an intuitive picker interface
  • Live Preview: See changes instantly on real components
  • WCAG Contrast Checker: Automatic accessibility validation with AA/AAA ratings
  • Preset Themes: Start with Dark, Light, High Contrast, or Colorblind-Friendly
  • Export Options: Download as CSS file or copy to clipboard
  • Real-time Validation: Ensures your theme meets accessibility standards

How to Use

  1. Visit the Theme Builder
  2. Start with a preset theme or customize from scratch
  3. Adjust colors using the color pickers
  4. Check the live preview to see your theme in action
  5. Verify accessibility with the contrast checker
  6. Export your theme as CSS or copy the code
  7. Add the custom CSS to your project

Dynamic Theme Switching

Allow users to switch between themes at runtime. Here's a simple implementation using JavaScript.

<!-- HTML -->
<link rel="stylesheet" href="./dark.css" id="theme-link">
<button onclick="toggleTheme()">Toggle Theme</button>

<!-- JavaScript -->
<script>
  function toggleTheme() {
    const themeLink = document.getElementById('theme-link');
    const currentTheme = themeLink.getAttribute('href');

    if (currentTheme.includes('dark')) {
      themeLink.setAttribute('href', './light.css');
      localStorage.setItem('theme', 'light');
    } else {
      themeLink.setAttribute('href', './dark.css');
      localStorage.setItem('theme', 'dark');
    }
  }

  // Load saved theme preference
  const savedTheme = localStorage.getItem('theme');
  if (savedTheme) {
    document.getElementById('theme-link')
      .setAttribute('href', `./${savedTheme}.css`);
  }
</script>

Try it now! Click the theme toggle button in the top right corner to see theme switching in action.

Best Practices

Do

  • Test themes in both light and dark modes
  • Ensure WCAG AA contrast standards (4.5:1)
  • Save user theme preferences
  • Use semantic token names
  • Document custom theme tokens

Don't

  • Use hardcoded color values
  • Override base component styles directly
  • Create themes with poor contrast
  • Forget to test on mobile devices
  • Mix multiple theming approaches
