Switch

iOS-style toggle switches for binary on/off options. Provide immediate visual feedback for settings and feature toggles.

Basic Switches

Basic switches provide instant on/off toggling. Use role="switch" for proper accessibility.

<label class="switch">
    <input type="checkbox" class="switch__input" role="switch" checked>
    <div class="switch__track">
        <div class="switch__thumb"></div>
    </div>
    <span class="switch__label">Enable notifications</span>
</label>

With Description

Add descriptions to provide context. Use switch-with-description and nest a switch__description span.

<label class="switch switch-with-description">
    <input type="checkbox" class="switch__input" role="switch">
    <div class="switch__track">
        <div class="switch__thumb"></div>
    </div>
    <span class="switch__label">
        Email notifications
        <span class="switch__description">Receive email updates</span>
    </span>
</label>
Label Position

Control label position with switch-label-left class. Default is label on the right.

<label class="switch switch-label-left">
    <input type="checkbox" class="switch__input" role="switch" checked>
    <div class="switch__track">
        <div class="switch__thumb"></div>
    </div>

</label>

Best Practices

When to Use

  • Settings and preferences - For toggling features on/off in settings panels
  • Instant changes - When changes take effect immediately without form submission
  • Binary states - For clear on/off, enabled/disabled choices
  • Visibility controls - Showing/hiding content or features

When NOT to Use

  • Form submissions - Use checkboxes for form data that requires submission
  • Multiple selections - Use checkboxes for selecting multiple items from a list
  • Single choice from many - Use radio buttons for mutually exclusive options
  • Destructive actions - Don't use switches for delete or irreversible actions

Design Guidelines

  • Clear labels - Use action-oriented labels that describe what the switch controls
  • Positive framing - Frame labels positively ("Enable" vs "Disable")
  • Add descriptions - Use descriptions to clarify what enabling a switch does
  • Visual feedback - Ensure smooth animations and clear on/off states

Accessibility

  • Use role="switch" - Add role="switch" to the input for proper ARIA semantics
  • Keyboard support - Ensure Space key toggles the switch
  • Focus indicators - Maintain visible focus states for keyboard navigation
  • State announcement - Screen readers should announce "on" or "off" state

Common Patterns

  • Settings panels - Group related switches by category with section headers
  • Feature toggles - Enable/disable app features with immediate effect
  • Privacy controls - Toggle visibility and data sharing preferences
  • Notification preferences - Control which notifications are enabled

Switch vs Checkbox

  • Use switches - For settings that take effect immediately (like iOS)
  • Use checkboxes - For form data, multi-select, or actions requiring submission
  • Context matters - In forms use checkboxes; in settings use switches
