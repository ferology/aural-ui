Radio Buttons

Single-selection radio button groups. Allow users to select exactly one option from a set of mutually exclusive choices.

Basic Radio Group
Select your plan

Use radio groups when users must select exactly one option. All radios in a group must share the same name attribute.

View Code 
<div class="radio-group">
    <div class="radio-group-label">Select your plan</div>
    <label class="radio">
        <input type="radio" name="plan" value="free" checked>
        <span>Free - $0/month</span>
    </label>
</div>
Sizes

Three size variants available: radio-sm, default, and radio-lg.

View Code 
<label class="radio radio-lg">
    <input type="radio" name="size" checked>
    <span>Large radio button</span>
</label>
With Description
Choose shipping method

Add descriptive text below labels to provide additional context for each option.

View Code 
<label class="radio radio-with-description">
    <input type="radio" name="shipping" value="express">
    <span>
        Express Shipping
        <span class="radio-description">Delivery in 2-3 days</span>
    </span>
</label>
Button Style

Button-style radios work well for view toggles and toolbar options. Use radio-group-buttons wrapper.

View Code 
<div class="radio-group radio-group-buttons">
    <label class="radio radio-button">
        <input type="radio" name="view" value="grid" checked>
        <span>Grid</span>
    </label>
</div>
Attached Buttons

Attached button radios create a segmented control appearance, ideal for toolbars and filters.

View Code 
<div class="radio-group radio-group-attached">
    <label class="radio radio-button">
        <input type="radio" name="align" value="left" checked>
        <span>Left</span>
    </label>
</div>
Card Style Selection

Card-style radios are excellent for pricing tiers, feature comparisons, or any selection requiring more visual emphasis.

Best Practices

When to Use

  • Mutually exclusive choices - When users must select exactly one option from a set
  • Small option sets - Best with 2-7 options; use dropdowns for longer lists
  • All options visible - When it's important for users to see all choices at once
  • Required selection - When a choice must be made before proceeding

When NOT to Use

  • Multiple selections - Use checkboxes when users can select more than one option
  • Many options - Use a select dropdown for 8+ options to save space
  • Binary toggles - Use a switch or single checkbox for simple on/off choices
  • Optional selection - If selecting nothing is valid, consider a different pattern

Organization

  • Use radio-group - Wrap related radios in a radio-group container
  • Logical order - Arrange options by frequency of use, alphabetically, or by value
  • Pre-select default - Always have one option selected by default when appropriate
  • Keep labels concise - Use short, scannable labels; add descriptions for clarity

Accessibility

  • Same name attribute - All radios in a group must share the same name attribute
  • Unique values - Each radio should have a unique value attribute
  • Keyboard navigation - Arrow keys navigate within group, Tab moves between groups
  • Label association - Wrap inputs in <label> tags for click targets
  • Group labels - Use radio-group-label or <fieldset>/<legend>

Common Patterns

  • Pricing tiers - Use card-style radios with visual hierarchy for plans
  • Shipping options - Include descriptions with delivery times and costs
  • View toggles - Use button-style or attached radios for UI view switches
  • Form sections - Use standard radios for questionnaires and surveys

const Radio = ({ name, value, checked, onChange, children, ...props }) => {
  return (
    <label className="radio">
      <input
        type="radio"
        name={name}
        value={value}
        className="radio__input"
        checked={checked}
        onChange={onChange}
        {...props}
      />
      <span className="radio__circle"></span>
      <span className="radio__label">{children}</span>
    </label>
  );
};

// Usage
const [selected, setSelected] = useState('option1');

<div className="radio-group">
  <Radio
    name="options"
    value="option1"
    checked={selected === 'option1'}
    onChange={(e) => setSelected(e.target.value)}
  >
    Option 1
  </Radio>
  <Radio
    name="options"
    value="option2"
    checked={selected === 'option2'}
    onChange={(e) => setSelected(e.target.value)}
  >
    Option 2
  </Radio>
</div>
<template>
  <label class="radio">
    <input
      type="radio"

      :value="value"
      class="radio__input"
      v-model="selected"
    />
    <span class="radio__circle"></span>
    <span class="radio__label">
      <slot />
    </span>
  </label>
</template>
