When to Use Cards

  • Content Grouping: Group related information together (user profiles, product listings, articles)
  • Dashboards: Display metrics, stats, and data visualizations
  • Feature Grids: Showcase features, services, or options
  • E-commerce: Product cards with images, descriptions, and prices
  • Media Galleries: Image or video collections with captions
  • List Views: Alternative to table rows for rich content

Card Anatomy

Cards typically consist of these sections (all optional):

  • Header: Title, subtitle, badges, or actions (optional)
  • Image: Visual content at the top (optional)
  • Body: Main content area (required)
  • Footer: Actions, links, or metadata (optional)

Accessibility

  • Use semantic HTML inside cards (<h3> for titles, <p> for content)
  • For clickable cards, wrap entire card in <a> or <button> with aria-label
  • Ensure sufficient color contrast for all text elements
  • Add keyboard navigation for interactive cards (tabindex="0")
  • Use descriptive alt text for images in .card-image
  • Don't rely solely on color to convey meaning - use text and icons

Best Practices

Layout Guidelines

  • Consistent Heights: In grids, try to maintain consistent card heights for visual harmony
  • Responsive Grids: Use auto-fill or auto-fit with minmax() for flexible grids
  • Spacing: Use consistent gap values (typically var(--space-4) or var(--space-6))
  • Max Width: For standalone cards, limit width to 400-600px for readability
  • Mobile First: Cards naturally stack on mobile, making them perfect for responsive designs

Content Guidelines

  • Keep card titles concise and descriptive (2-5 words)
  • Limit body text to 2-3 sentences for scanability
  • Use clear, action-oriented button labels
  • Group related actions in the footer
  • Prioritize the most important information at the top

Visual Design

  • Maintain consistent padding across all cards in a grid
  • Use images with consistent aspect ratios in card grids
  • Limit the number of actions per card (2-3 maximum)
  • Use subtle hover effects on interactive cards (.card-hover)
  • Ensure cards have sufficient contrast with the background

Responsive Grid Pattern

Use CSS Grid for responsive card layouts:

  • repeat(auto-fill, minmax(300px, 1fr)) - Fill available space, wrap to new rows
  • repeat(auto-fit, minmax(300px, 1fr)) - Stretch cards to fill space
  • Adjust minimum width based on card content (200-350px typical)
  • Use consistent gap values for visual rhythm

const Card = ({ title, badge, children, actions }) => {
  return (
    <div className="card">
      {(title || badge) && (
        <div className="card-header">
          {title && <h3 className="card-title">{title}</h3>}
          {badge && <span className={`badge badge-${badge.variant}`}>{badge.text}</span>}
        </div>
      )}
      <div className="card-body">
        {children}
      </div>
      {actions && (
        <div className="card-footer">
          {actions}
        </div>
      )}
    </div>
  );
};

// Usage
<Card
  title="Card Title"
  badge={{ variant: 'success', text: 'Active' }}
  actions={<button className="btn btn-primary btn-sm">Action</button>}
>
  <p>Card content goes here</p>
</Card>
<template>
  <div class="card">
    <div v-if="title || badge" class="card-header">
      <h3 v-if="title" class="card-title">{{ title }}</h3>
      <span v-if="badge" :class="`badge badge-${badge.variant}`">
        {{ badge.text }}
      </span>
    </div>
    <div class="card-body">
      <slot />
    </div>
    <div v-if="$slots.actions" class="card-footer">
      <slot name="actions" />
    </div>
  </div>
</template>

<script>
export default {
  props: {
    title: String,
    badge: Object
  }
}
</script>
