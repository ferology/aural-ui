When to Use Badges

  • Status Indicators: Show order status, user status, or process states
  • Counts & Numbers: Display notification counts, unread messages, or quantities
  • Categories & Tags: Label content with topics, types, or classifications
  • Metadata: Show version numbers, dates, or other supplementary information
  • Feature Flags: Highlight new, beta, or experimental features

Accessibility

  • Use semantic color variants to convey meaning visually
  • For critical information, supplement color with text or icons
  • Keep badge text concise and clear (2-3 words maximum)
  • Ensure sufficient color contrast (badges meet WCAG AA standards)
  • For notification badges, use aria-label on parent elements to announce counts to screen readers
  • Mark visual-only badges with aria-hidden="true" when information is conveyed elsewhere

Best Practices

Color Usage

  • Primary: Default state or brand-related labels
  • Success: Positive status (Active, Complete, Approved)
  • Error: Negative status (Error, Failed, Rejected)
  • Warning: Cautionary status (Pending, Draft, Warning)
  • Info: Informational status (New, Beta, Info)

Sizing

  • Small: Compact size for tight spaces or inline text
  • Default: Standard size for most use cases
  • Large: Larger size for emphasis or standalone labels

Content Guidelines

  • Keep text short and scannable (1-2 words ideal)
  • Use sentence case for readability, unless all-caps is your brand style
  • Avoid punctuation in badges
  • For counts, use abbreviations for large numbers (1K, 10K, 1M)

const Badge = ({ children, variant = 'primary', size, icon }) => {
  const classNames = [
    'badge',
    `badge-${variant}`,
    size && `badge-${size}`
  ].filter(Boolean).join(' ');

  return (
    <span className={classNames}>
      {icon && <i data-lucide={icon} />}
      {children}
    </span>
  );
};

// Usage
<Badge variant="primary">Primary</Badge>
<Badge variant="success" icon="check">Verified</Badge>
<Badge variant="warning" size="sm">Warning</Badge>
<template>
  <span :class="badgeClasses">
    <i v-if="icon" :data-lucide="icon" />
    <slot />
  </span>
</template>

<script>
export default {
  props: {
    variant: { type: String, default: 'primary' },
    size: String,
    icon: String
  },
  computed: {
    badgeClasses() {
      return [
        'badge',
        `badge-${this.variant}`,
        this.size && `badge-${this.size}`
      ].filter(Boolean).join(' ');
    }
  }
}
</script>

<!-- Usage -->
<Badge variant="primary">Primary</Badge>
<Badge variant="success" icon="check">Verified</Badge>
<script>
  export let variant = 'primary';
  export let size = '';
  export let icon = '';

  $: badgeClasses = [
    'badge',
    `badge-${variant}`,
    size && `badge-${size}`
  ].filter(Boolean).join(' ');
</script>

<span class={badgeClasses}>
  {#if icon}
    <i data-lucide={icon} />
  {/if}
  <slot />
</span>
