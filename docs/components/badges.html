Badges

Small, versatile status indicators and labels for displaying counts, statuses, and categories. Perfect for notifications, tags, and metadata.

Semantic Variants
Primary Secondary Success Error Warning Info

Choose the appropriate semantic variant to convey meaning:

  • .badge-primary - Default state or brand-related labels
  • .badge-secondary - Alternative or less prominent labels
  • .badge-success - Positive status (Active, Complete, Approved)
  • .badge-error - Negative status (Error, Failed, Rejected)
  • .badge-warning - Cautionary status (Pending, Draft, Warning)
  • .badge-info - Informational status (New, Beta, Info)
Sizes
Small Default Large

Three sizes to fit different contexts:

  • .badge-sm - Compact size for tight spaces or inline text
  • Default - Standard size for most use cases
  • .badge-lg - Larger size for emphasis or standalone labels

Best Practices

When to Use Badges

  • Status Indicators: Show order status, user status, or process states
  • Counts & Numbers: Display notification counts, unread messages, or quantities
  • Categories & Tags: Label content with topics, types, or classifications
  • Metadata: Show version numbers, dates, or other supplementary information
  • Feature Flags: Highlight new, beta, or experimental features

Accessibility

  • Use semantic color variants to convey meaning visually
  • For critical information, supplement color with text or icons
  • Keep badge text concise and clear (2-3 words maximum)
  • Ensure sufficient color contrast (badges meet WCAG AA standards)
  • For notification badges, use aria-label on parent elements to announce counts to screen readers

Design Guidelines

  • Be Concise: Keep text short - one or two words is ideal
  • Be Consistent: Use the same badge variant for similar statuses across your app
  • Don't Overuse: Too many badges create visual noise and reduce their effectiveness
  • Consider Context: Use smaller badges for inline text, larger ones for standalone labels
  • Pair with Icons: Icons enhance understanding, especially for users with color blindness

Common Patterns

Status Tables: Use badges in table cells to show record status

<td>
  <span class="badge badge-success">Active</span>
</td>

Notification Counts: Position badges on icons or buttons

<div style="position: relative">
  <button class="btn btn-ghost btn-icon">
    <i data-lucide="bell"></i>
  </button>
  <span class="badge badge-error badge-sm"
        style="position: absolute; top: -4px; right: -4px">
    3
  </span>
</div>

Content Tags: Group related badges for categorization

<div style="display: flex; gap: var(--space-2)">
  <span class="badge badge-secondary">JavaScript</span>
  <span class="badge badge-secondary">Tutorial</span>
</div>
