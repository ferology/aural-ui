When to Use Avatars

  • User Identification: Display user profiles, account information, and identity
  • Comments & Posts: Attribute content to specific users in social feeds or discussions
  • Team Members: Show team rosters, collaborators, or project participants
  • User Lists: Enhance tables, lists, or directories with visual identification
  • Activity Feeds: Show who performed actions in activity logs or timelines
  • Navigation: Display current user in navigation bars or dropdown menus

Accessibility

  • Always provide meaningful alt text for avatar images describing the person
  • For clickable avatars, use semantic HTML (button or anchor tags) with descriptive aria-label attributes
  • Status indicators are visual only - include status information in nearby text or aria-label
  • Ensure sufficient color contrast for initials on background (meets WCAG AA standards)
  • Avatar groups maintain proper focus order for keyboard navigation
  • For decorative avatars without functional purpose, use alt="" on images

Best Practices

Size Selection

  • XS (24px): Inline mentions, compact lists, or very tight spaces
  • SM (32px): Table rows, comment threads, or activity feeds
  • Default (40px): Standard size for most UI contexts
  • LG (48px): User profiles, cards, or prominent displays
  • XL (64px): Profile headers, settings pages, or hero sections

Image Guidelines

  • Use square images with 1:1 aspect ratio for best results
  • Minimum recommended resolution: 200x200px for crisp display at all sizes
  • Optimize images for web to reduce load times
  • Provide fallback initials when images fail to load
  • Consider using WebP format for better compression and quality

Initials Display

  • Show 1-2 characters maximum (typically first and last name initials)
  • Use uppercase letters for consistency
  • Consider generating unique background colors per user for variety
  • Ensure text remains readable on all background colors

Status Indicators

  • Online: Green indicator for active/available users
  • Busy: Red indicator for do-not-disturb status
  • Away: Yellow indicator for idle or temporarily away
  • Offline: Gray indicator for signed out or unavailable
  • Update status in real-time for messaging and collaboration apps
  • Always supplement visual status with text for accessibility

Mobile Considerations

  • Clickable avatars automatically meet 44x44px touch target requirements on mobile devices
  • Status indicators are slightly larger on touch devices for better visibility
  • Avatar groups maintain adequate spacing for comfortable interaction
  • Consider using larger avatar sizes on mobile for better visibility

const Avatar = ({ src, alt, initials, size, status, onClick }) => {
  const classNames = [
    'avatar',
    size && `avatar-${size}`,
    !src && initials && 'avatar-initials',
    onClick && 'avatar-clickable'
  ].filter(Boolean).join(' ');

  return (
    <div className={classNames} onClick={onClick}>
      {src ? (
        <img src={src} alt={alt} />
      ) : (
        initials
      )}
      {status && (
        <span className={`avatar-status avatar-status-${status}`} />
      )}
    </div>
  );
};

<template>
  <div :class="avatarClasses" @click="handleClick">
    <img v-if="src" :src="src" :alt="alt" />
    <template v-else>{{ initials }}</template>
    <span
      v-if="status"
      :class="`avatar-status avatar-status-${status}`"
    />
  </div>
</template>

<script>
export default {
  props: {
    src: String,
    alt: String,
    initials: String,
    size: String,
    status: String
  },
  computed: {
    avatarClasses() {
      return [
        'avatar',
        this.size && `avatar-${this.size}`,
        !this.src && this.initials && 'avatar-initials',
        this.$listeners.click && 'avatar-clickable'
      ].filter(Boolean).join(' ');
    }
  },
  methods: {
    handleClick() {
      this.$emit('click');
    }
  }
}
</script>

<script>
  import { createEventDispatcher } from 'svelte';

  export let src = '';
  export let alt = '';
  export let initials = '';
  export let size = '';
  export let status = '';

  const dispatch = createEventDispatcher();

  $: avatarClasses = [
    'avatar',
    size && `avatar-${size}`,
    !src && initials && 'avatar-initials',
    $$listeners.click && 'avatar-clickable'
  ].filter(Boolean).join(' ');

  function handleClick() {
    dispatch('click');
  }
</script>

<div class={avatarClasses} on:click={handleClick}>
  {#if src}
    <img {src} {alt} />
  {:else}
    {initials}
  {/if}
  {#if status}
    <span class="avatar-status avatar-status-{status}" />
  {/if}
</div>
