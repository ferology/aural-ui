Tooltips

Contextual help text that appears on hover or focus. Tooltips provide additional information without cluttering the interface.

Tooltips can appear in four positions relative to the trigger element:

  • .tooltip-top - Above the element (default)
  • .tooltip-right - To the right of the element
  • .tooltip-bottom - Below the element
  • .tooltip-left - To the left of the element

Usage: Wrap your element in .tooltip and add the position modifier class. The tooltip text goes in a .tooltip-text element.

Additional information about this field

Tooltips work on any inline or inline-block element. Common use cases:

  • Buttons: Explain actions or show keyboard shortcuts
  • Icon Buttons: Label icon-only buttons for clarity
  • Links: Preview link destinations or provide context
  • Badges: Explain status meanings or show details
  • Info Icons: Provide help text for form fields or features

Good tooltip content:

  • Concise (5-10 words ideal, max 2 lines)
  • Adds meaningful context not already visible
  • Uses plain language
  • Includes keyboard shortcuts when relevant

Avoid:

  • Repeating visible label text
  • Long paragraphs (use popovers or modals instead)
  • Critical information users need to see
  • Interactive elements inside tooltips

Best Practices

When to Use Tooltips

  • Icon Labels: Label icon-only buttons that lack text labels
  • Keyboard Shortcuts: Show keyboard shortcuts for actions
  • Truncated Text: Reveal full text that's been truncated with ellipsis
  • Additional Context: Provide brief explanatory notes
  • Status Clarification: Explain badges or status indicators
  • Form Help: Offer hints for form fields (pair with info icons)

When NOT to Use Tooltips

  • Critical Information: Never hide essential info in tooltips - users might not discover it
  • Long Content: Use popovers, modals, or expandable sections for lengthy explanations
  • Touch Interfaces: Tooltips don't work well on mobile/touch devices (provide alternative access)
  • Complex Interactions: Don't put links, buttons, or forms inside tooltips
  • Repeating Labels: Don't tooltip text that's already visible

Accessibility

  • Keyboard Access: Tooltips should appear on both hover and focus
  • Screen Readers: Use aria-label or aria-describedby for screen reader users
  • Dismissible: Users should be able to dismiss tooltips with Escape key
  • Touch Support: Consider tap-to-show on touch devices, or provide alternative access
  • Color Independence: Don't rely solely on color to convey information in tooltips

Design Guidelines

  • Timing: Show after ~500ms hover delay to avoid accidental triggers
  • Positioning: Choose positions that keep tooltips visible (avoid edges)
  • Contrast: Ensure text is readable against tooltip background
  • Animation: Use subtle fade-in animations (included by default)
  • Consistency: Use same position throughout similar contexts

Common Patterns

Icon Button Tooltip:

<div class="tooltip tooltip-top">
  <button class="btn btn-ghost btn-icon" aria-label="Settings">
    <i data-lucide="settings"></i>
  </button>
  <span class="tooltip-text">Settings</span>
</div>

Action with Keyboard Shortcut:

<div class="tooltip tooltip-top">
  <button class="btn btn-primary">
    <i data-lucide="save"></i>
    Save
  </button>
  <span class="tooltip-text">Save changes (Ctrl+S)</span>
</div>

Form Field Help:

<label>
  Password
  <div class="tooltip tooltip-right" style="display: inline-block">
    <i data-lucide="info" style="width: 16px"></i>
    <span class="tooltip-text">
      Must be at least 8 characters
    </span>
  </div>
</label>
<input type="password" class="input">

Status Badge with Detail:

<div class="tooltip tooltip-top">
  <span class="badge badge-success">Verified</span>
  <span class="tooltip-text">
    Account verified on Jan 15, 2026
  </span>
</div>

Mobile Considerations

Tooltips are problematic on touch devices since there's no hover state. Consider these alternatives:

  • Visible Labels: Show labels directly instead of hiding in tooltips
  • Tap to Reveal: Show tooltip on tap/click (requires JavaScript)
  • Info Modals: Use modals or sheets for detailed help on mobile
  • Progressive Disclosure: Use expandable sections or accordions
