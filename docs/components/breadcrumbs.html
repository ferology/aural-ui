Framework Examples

<nav class="breadcrumb" aria-label="Breadcrumb">
    <ol>
        <li><a href="/">Home</a></li>
        <li><a href="/products">Products</a></li>
        <li><a href="/products/category">Category</a></li>
        <li aria-current="page">Item</li>
    </ol>
</nav>
const Breadcrumb = ({ items }) => {
  return (
    <nav className="breadcrumb" aria-label="Breadcrumb">
      <ol>
        {items.map((item, index) => (
          <li key={index}>
            {index === items.length - 1 ? (
              <span aria-current="page">{item.label}</span>
            ) : (
              <a href={item.href}>{item.label}</a>
            )}
          </li>
        ))}
      </ol>
    </nav>
  );
};

// Usage
const items = [
  { label: 'Home', href: '/' },
  { label: 'Products', href: '/products' },
  { label: 'Category', href: '/products/category' },
  { label: 'Item' }
];

<Breadcrumb items={items} />
<template>
  <nav class="breadcrumb" aria-label="Breadcrumb">
    <ol>
      <li v-for="(item, index) in items" :key="index">
        <a v-if="index !== items.length - 1" :href="item.href">
          {{ item.label }}
        </a>
        <span v-else aria-current="page">{{ item.label }}</span>
      </li>
    </ol>
  </nav>
</template>

<script>
export default {
  props: {
    items: {
      type: Array,
      required: true
    }
  }
}
</script>

<!-- Usage -->
<Breadcrumb :items="[
  { label: 'Home', href: '/' },
  { label: 'Products', href: '/products' },
  { label: 'Category', href: '/products/category' },
  { label: 'Item' }
]" />

Best Practices

When to Use

  • Use breadcrumbs for sites with multiple levels of navigation hierarchy
  • Show breadcrumbs on pages that are 2 or more levels deep from the homepage
  • Use in e-commerce sites to show product category paths
  • Use in documentation sites to show document structure
  • Use in file management interfaces to show folder hierarchy

When Not to Use

  • Don't use breadcrumbs on single-level websites or simple structures
  • Don't use as a replacement for primary navigation
  • Don't use when pages don't have a clear hierarchical structure
  • Don't include the current page as a clickable link

Accessibility

  • Always include aria-label="Breadcrumb" on the nav element
  • Use aria-current="page" on the current page item
  • Use semantic <nav> and <ol> elements
  • Ensure sufficient color contrast for links (WCAG AA: 4.5:1 minimum)
  • Make sure links have visible focus indicators for keyboard navigation
  • Keep link text clear and descriptive

Content Guidelines

  • Keep breadcrumb labels short and clear (1-3 words)
  • Use sentence case for labels, not title case
  • Start with the homepage or top-level category
  • Include all levels in the hierarchy
  • The last item should be the current page (not clickable)
  • Use collapsed variant for very deep hierarchies (5+ levels)

Separator Guidelines

  • Default slash (/) - Universal, works for most contexts
  • .breadcrumb-chevron - Modern, popular choice for web applications
  • .breadcrumb-arrow - Good for file paths and linear flows
  • .breadcrumb-dash - Subtle, works well with minimal designs
  • .breadcrumb-dot - Light visual weight, good for compact spaces

Placement

  • Place breadcrumbs near the top of the page, below the main navigation
  • Position above the page title or content heading
  • Align breadcrumbs to the left of the page content
  • Maintain consistent placement across all pages
