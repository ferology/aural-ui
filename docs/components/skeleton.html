            <div style="padding: var(--space-4);">
                <div style="display: flex; gap: var(--space-6); align-items: center; margin-bottom: var(--space-4);">

                        <div class="skeleton skeleton-text" style="width: 40%;"></div>

                    </div>
                </div>
                <div class="skeleton skeleton-text" style="width: 100%; margin-bottom: var(--space-2);"></div>
                <div class="skeleton skeleton-text" style="width: 90%;"></div>
            </div>
        </div>
    `;

    // Load actual content after 2 seconds
    setTimeout(() => {
        container.innerHTML = `<!-- Actual content -->`;
    }, 2000);
}
</script>

When to Use Skeleton

  • Content Loading: Display while fetching data from APIs or databases
  • Initial Page Load: Show placeholders during first paint for better perceived performance
  • Progressive Loading: Indicate where content will appear as it loads incrementally
  • Lazy Loading: Show placeholders for images and components that load on scroll
  • Complex Layouts: Preview the structure of cards, lists, profiles, or tables before data arrives

Accessibility

  • Add aria-busy="true" to the container while loading
  • Include visually hidden text like "Loading content..." for screen readers
  • Remove aria-busy and update aria-live regions when content loads
  • Don't rely solely on animation - ensure skeleton shapes clearly indicate content structure
  • Consider reduced motion preferences - provide non-animated alternatives
  • Ensure sufficient contrast between skeleton and background colors

Best Practices

Shape & Structure

  • Match skeleton shapes to actual content layout as closely as possible
  • Use skeleton-text for text lines
  • Use skeleton-circle for avatars and circular elements
  • Use skeleton-rect for images, buttons, and rectangular content
  • Vary line widths slightly to create more natural appearance

Timing

  • Show skeletons immediately when loading begins
  • Display for minimum 200-300ms to avoid flash of loading state
  • Don't show skeletons for very fast loads (under 200ms)
  • Replace with actual content smoothly without jarring transitions
  • Consider fade-in animation when content replaces skeleton

Animation

  • Use subtle shimmer/pulse animation to indicate active loading
  • Keep animation speed moderate (1-2 second cycle)
  • Ensure animation respects prefers-reduced-motion setting
  • Don't make animation too distracting or bright

Content Patterns

  • Create reusable skeleton components for common patterns (cards, lists, profiles)
  • Show the same number of skeleton items as typical content count
  • Include all major UI elements (avatars, buttons, actions) in skeleton
  • Maintain proper spacing and alignment with final content

When Not to Use

  • For very quick operations (under 200ms)
  • During background refresh of already-visible content
  • For critical actions where spinner may be more appropriate
  • When actual loading time is unknown or highly variable
