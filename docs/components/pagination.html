Framework Examples

<nav class="pagination" aria-label="Pagination">
    <a href="#" class="pagination-prev" aria-label="Go to previous page">
        <i data-lucide="chevron-left"></i>
        Previous
    </a>
    <a href="#" class="pagination-item" aria-label="Go to page 1">1</a>
    <a href="#" class="pagination-item pagination-active" aria-current="page">2</a>
    <a href="#" class="pagination-item" aria-label="Go to page 3">3</a>
    <a href="#" class="pagination-next" aria-label="Go to next page">
        Next
        <i data-lucide="chevron-right"></i>
    </a>
</nav>
const Pagination = ({ currentPage, totalPages, onPageChange }) => {
  return (
    <nav className="pagination" aria-label="Pagination">
      <button
        className="pagination-prev"
        onClick={() => onPageChange(currentPage - 1)}
        disabled={currentPage === 1}
        aria-label="Go to previous page"
      >
        <i data-lucide="chevron-left" />
        Previous
      </button>

      {Array.from({ length: totalPages }, (_, i) => i + 1).map(page => (
        <button
          key={page}
          className={`pagination-item ${page === currentPage ? 'pagination-active' : ''}`}
          onClick={() => onPageChange(page)}
          aria-current={page === currentPage ? 'page' : undefined}
          aria-label={`Go to page ${page}`}
        >
          {page}
        </button>
      ))}

      <button
        className="pagination-next"
        onClick={() => onPageChange(currentPage + 1)}
        disabled={currentPage === totalPages}
        aria-label="Go to next page"
      >
        Next
        <i data-lucide="chevron-right" />
      </button>
    </nav>
  );
};

// Usage
const [page, setPage] = useState(1);
<Pagination currentPage={page} totalPages={10} onPageChange={setPage} />
<template>
  <nav class="pagination" aria-label="Pagination">
    <button
      class="pagination-prev"
      @click="$emit('pageChange', currentPage - 1)"
      :disabled="currentPage === 1"
      aria-label="Go to previous page"
    >
      <i data-lucide="chevron-left" />
      Previous
    </button>

    <button
      v-for="page in totalPages"
      :key="page"
      :class="['pagination-item', { 'pagination-active': page === currentPage }]"
      @click="$emit('pageChange', page)"
      :aria-current="page === currentPage ? 'page' : undefined"
      :aria-label="`Go to page ${page}`"
    >
      {{ page }}
    </button>

    <button
      class="pagination-next"
      @click="$emit('pageChange', currentPage + 1)"
      :disabled="currentPage === totalPages"
      aria-label="Go to next page"
    >
      Next
      <i data-lucide="chevron-right" />
    </button>
  </nav>
</template>

<script>
export default {
  props: {
    currentPage: { type: Number, required: true },
    totalPages: { type: Number, required: true }
  },
  emits: ['pageChange']
}
</script>
<script>
  import { Pagination } from '@aural-ui/svelte';

  let currentPage = 1;
  const totalPages = 10;

  function handlePageChange(e) {
    currentPage = e.detail;
    console.log('Page changed to:', currentPage);
  }
</script>

  on:pageChange={handlePageChange}

Best Practices

When to Use

  • Use pagination for large datasets that would be overwhelming on a single page
  • Use when users need to browse through sequential content
  • Use at the bottom of data tables, search results, or product listings
  • Use when the total number of items is known
  • Consider pagination for lists with more than 25-50 items

When Not to Use

  • Don't use for small datasets (less than 25 items) - show all at once
  • Don't use for real-time data feeds - use infinite scroll instead
  • Don't use when users need to find specific items - provide search/filter first
  • Don't use when content doesn't have a natural order

Accessibility

  • Always include aria-label="Pagination" on the nav element
  • Use aria-current="page" on the active page item
  • Add descriptive aria-label attributes to all page links
  • Use aria-disabled="true" on disabled prev/next buttons
  • Add aria-hidden="true" to ellipsis elements
  • Ensure minimum touch target size of 44x44px for mobile
  • Provide clear keyboard focus indicators
  • Make sure disabled states have sufficient visual differentiation

Content Guidelines

  • Always show the current page as non-clickable (active state)
  • Disable prev button on first page, next button on last page
  • Show first and last page numbers when using ellipsis
  • Display 1-2 pages before and after current page
  • Use ellipsis (...) to indicate hidden pages
  • Consider showing page count info: "Page 2 of 10" or "Showing 11-20 of 200"

Variant Guidelines

  • Default - Standard bordered buttons, works in most contexts
  • .pagination-rounded - Fully rounded, modern appearance
  • .pagination-simple - Borderless, minimal visual weight
  • .pagination-compact - Attached buttons, space-efficient
  • .pagination-sm - Use for tables, sidebars, or compact layouts
  • .pagination-lg - Use for prominent navigation or hero sections

UX Best Practices

  • Keep pagination visible and easy to find
  • Maintain consistent position across pages (usually bottom of content)
  • Preserve user filters and sort settings when navigating pages
  • Show loading states when fetching new page data
  • Consider "Load More" button as alternative for mobile
  • Update URL with page parameter for bookmarking and sharing
  • Scroll to top of content when page changes
  • Consider adding "Jump to page" input for large page counts (20+ pages)

Performance

  • Implement server-side pagination for large datasets
  • Use appropriate page sizes (10-50 items per page)
  • Consider infinite scroll for mobile experiences
  • Cache previously loaded pages when possible
  • Preload adjacent pages for faster navigation
