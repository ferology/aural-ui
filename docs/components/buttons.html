Buttons

Interactive buttons with multiple variants, sizes, and states.

API Reference

Class Description Type
.btn Base button class Required
.btn-primary Primary action button with brand color Optional
.btn-secondary Secondary action button Optional
.btn-danger Destructive action button Optional
.btn-ghost Subtle button with minimal styling Optional
.btn-sm Small button variant Modifier
.btn-lg Large button variant Modifier
disabled Disabled state attribute Modifier

Framework Examples

<button class="btn btn-primary">Click me</button>
<button class="btn btn-secondary">Cancel</button>
<button class="btn btn-danger">Delete</button>
const Button = ({ children, variant = 'primary', ...props }) => {
  return (
    <button className={`btn btn-${variant}`} {...props}>
      {children}
    </button>
  );
};

// Usage
<Button variant="primary">Click me</Button>
<Button variant="secondary">Cancel</Button>
<Button variant="danger">Delete</Button>
<template>
  <button :class="`btn btn-${variant}`">
    <slot />
  </button>
</template>

<script>
export default {
  props: {
    variant: { type: String, default: 'primary' }
  }
}
</script>

Icon-only buttons are perfect for toolbars and compact interfaces. Use the .btn-icon class to create square buttons with centered icons. Available in all sizes:

  • .btn-icon.btn-sm - 36×36px (small)
  • .btn-icon - 44×44px (default)
  • .btn-icon.btn-lg - 52×52px (large)

Best Practices

When to Use

  • Use .btn-primary for the main action on a page
  • Use .btn-secondary for secondary actions
  • Use .btn-danger for destructive actions (delete, remove, etc.)
  • Use .btn-ghost for tertiary actions or minimal UI

Accessibility

  • Always provide descriptive button text that clearly indicates the action
  • For icon-only buttons, include a title or aria-label attribute for screen readers
  • Ensure sufficient color contrast between button text and background
  • Use the disabled attribute to disable buttons, not just visual styling
  • Provide visual feedback for loading states using .btn-loading and .btn-spinner

Button Sizing

  • .btn-sm - Use for compact interfaces and tables
  • Default size - Use for most UI actions
  • .btn-lg - Use for prominent calls-to-action or hero sections

Icon Guidelines

  • Icons should enhance, not replace, button text for primary actions
  • Icon-only buttons are best for secondary actions in toolbars
  • Keep icons simple and universally recognizable
  • Always ensure icon-only buttons have proper accessibility labels
