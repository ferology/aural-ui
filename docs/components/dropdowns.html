Dropdowns

Reveal contextual menus and actions on demand. Dropdowns display a list of options or actions when triggered, providing access to secondary functions without cluttering the interface.

Best Practices

When to Use Dropdowns

  • Action Menus: Provide multiple actions for an item (edit, share, delete)
  • User Menus: Account settings, profile, sign out options
  • Bulk Actions: Actions on selected table rows or list items
  • View Options: Filter, sort, or display options for content
  • Navigation Overflow: Secondary navigation items that don't fit in primary nav

When NOT to Use Dropdowns

  • Form Inputs: Use a <select> element or custom select component instead
  • Single Action: If there's only one option, use a direct button instead
  • Primary Actions: Don't hide critical actions in dropdowns - make them visible
  • Many Options: For 10+ items, consider a modal or dedicated page
  • Mobile Primary Nav: Use a drawer or full-screen menu for mobile navigation

Menu Organization

  • Logical Grouping: Group related actions together with dividers
  • Frequent First: Place most common actions at the top
  • Destructive Last: Place delete/remove actions at the bottom, separated by a divider
  • Color for Danger: Use red/error color for destructive actions to warn users
  • Consistent Order: Keep menu item order consistent across your application

Accessibility

  • Use semantic <button> elements for triggers
  • Add aria-haspopup="true" and aria-expanded to dropdown buttons
  • Support keyboard navigation: Tab to button, Enter/Space to open, Arrow keys to navigate items, Esc to close
  • Use aria-label on icon-only buttons (e.g., "More options")
  • Ensure menu items have visible focus states
  • Close dropdown when clicking outside or pressing Escape
  • Return focus to trigger button when closing

Positioning

  • Default: Menu opens below the trigger, aligned to the left
  • Right Aligned: Use .dropdown-menu-right for dropdowns near right edge
  • Auto-flip: Menu should flip direction if it would overflow the viewport
  • Table Rows: Right-align menus in table rows to prevent overflow
  • Z-index: Ensure dropdowns appear above other content (z-index: 1000+)

Mobile Considerations

  • Touch Targets: Ensure menu items are at least 44px tall for easy tapping
  • Full Width: Consider full-width dropdowns on small screens
  • Bottom Sheets: For mobile, consider using bottom sheets instead of dropdowns
  • Close on Scroll: Close dropdowns when user scrolls the page

Common Patterns

Simple Action Menu:

<div class="dropdown">
  <button class="btn btn-secondary dropdown-toggle">
    Actions <i data-lucide="chevron-down"></i>
  </button>
  <div class="dropdown-menu">
    <a href="#" class="dropdown-item">Edit</a>
    <a href="#" class="dropdown-item">Share</a>
    <div class="dropdown-divider"></div>
    <a href="#" class="dropdown-item" style="color: var(--color-error);">
      Delete
    </a>
  </div>
</div>
