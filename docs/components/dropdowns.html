Dropdowns

Reveal contextual menus and actions on demand. Dropdowns display a list of options or actions when triggered, providing access to secondary functions without cluttering the interface.

Best Practices

When to Use Dropdowns

  • Action Menus: Provide multiple actions for an item (edit, share, delete)
  • User Menus: Account settings, profile, sign out options
  • Bulk Actions: Actions on selected table rows or list items
  • View Options: Filter, sort, or display options for content
  • Navigation Overflow: Secondary navigation items that don't fit in primary nav

Dropdown Organization

  • Logical Grouping: Group related actions together with dividers
  • Frequent First: Place most common actions at the top
  • Destructive Last: Place delete/remove actions at the bottom, separated by a divider
  • Color for Danger: Use red/error color for destructive actions to warn users
  • Consistent Order: Keep menu item order consistent across your application

Accessibility

  • Use semantic <button> elements for triggers
  • Add aria-expanded attribute to show open/closed state
  • Support keyboard navigation: Enter/Space to open, Arrow keys to navigate, Esc to close
  • Use aria-label on icon-only buttons (e.g., "More options")
  • Each dropdown must have a unique id attribute

Common Patterns

Simple Action Menu:

<div class="dropdown" id="actions">
  <button class="btn btn-secondary dropdown-trigger">
    Actions
  </button>
  <div class="dropdown-menu">
    <a class="dropdown-item">Edit</a>
    <a class="dropdown-item">Share</a>
    <div class="dropdown-divider"></div>
    <a class="dropdown-item">Delete</a>
  </div>
</div>

  <button class="btn btn-ghost btn-icon dropdown-trigger">
