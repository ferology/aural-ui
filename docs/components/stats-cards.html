When to Use Stats Cards

  • Dashboard Metrics: Display key performance indicators on admin or analytics dashboards
  • Performance Tracking: Show growth trends, conversions, and business metrics over time
  • User Analytics: Present user engagement data, session statistics, and activity metrics
  • Resource Monitoring: Track storage usage, CPU performance, and system resources
  • Financial Data: Display revenue, expenses, and other monetary statistics

Accessibility

  • Use semantic HTML structure with proper heading hierarchy
  • Mark decorative icons with aria-hidden="true"
  • For progress bars, include role="progressbar" with aria-valuenow, aria-valuemin, and aria-valuemax attributes
  • Ensure sufficient color contrast between text and backgrounds (WCAG AA minimum)
  • Don't rely solely on color to convey meaning - use text labels and icons
  • Provide context with descriptive labels and comparison periods

Best Practices

Layout & Composition

  • Use responsive grids that adapt to different screen sizes
  • Group related metrics together for easier scanning
  • Prioritize the most important metrics with larger cards or prominent placement
  • Maintain consistent spacing between cards for visual rhythm

Data Presentation

  • Keep values clear and prominently displayed
  • Use appropriate number formatting (K for thousands, M for millions)
  • Include comparison periods for context (vs last month, vs last year)
  • Show trends with directional indicators when relevant
  • Add progress bars for metrics with known limits (storage, completion)

Color Usage

  • Primary: Default metrics and brand-related data
  • Success: Positive metrics, growth, achievements
  • Warning: Cautionary metrics, approaching limits
  • Error: Negative metrics, errors, critical issues

Icons

  • Use icons that clearly represent the metric type
  • Keep icons simple and recognizable
  • Maintain consistent icon size across all cards
  • Icons should enhance understanding, not be purely decorative
