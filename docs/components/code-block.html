When to Use Code Blocks

Use code blocks to display programming code and technical content:

  • Documentation: Show code examples in technical documentation
  • Tutorials: Display code snippets in learning materials
  • API references: Show request/response examples
  • Configuration: Display config files and settings
  • Terminal commands: Show CLI commands and output
  • Inline code: Reference code elements within text

Best Practices

Code Presentation

  • Always specify the language label for better context
  • Keep code examples concise and focused on the concept being demonstrated
  • Include copy buttons for user convenience
  • Remove unnecessary comments unless they add educational value
  • Use proper indentation and formatting

Line Numbers

  • Use line numbers for longer code blocks (more than 5 lines)
  • Enable line numbers when referencing specific lines in documentation
  • Omit line numbers for short snippets or terminal commands

Highlighting

  • Use --highlight to draw attention to important lines
  • Use --success to show correct implementations
  • Use --error to highlight mistakes or anti-patterns
  • Avoid over-highlighting - use sparingly for maximum impact

Context and Labels

  • Always label the language or file type
  • Provide brief descriptions before code blocks
  • For configuration files, use the filename as the label
  • Group related code blocks together logically

Inline Code

  • Use inline code for short references: variable names, function names, commands
  • Don't use inline code for long strings or multi-line code
  • Use inline code consistently within a paragraph

Accessibility

Ensure code blocks are accessible to all users:

  • Use semantic HTML: <pre> and <code> elements
  • Ensure sufficient color contrast for code text (at least 4.5:1)
  • Make copy buttons keyboard accessible with proper focus states
  • Add aria-label to copy buttons: "Copy code"
  • Consider adding language information for screen readers
  • For highlighted lines, ensure color is not the only distinguisher
  • Ensure horizontal scrolling is keyboard accessible

Mobile Considerations

  • Enable horizontal scrolling for wide code blocks
  • Consider using smaller font sizes on mobile (use --sm modifier)
  • Ensure copy buttons are large enough for touch (minimum 44×44px)
  • Test code blocks on various screen sizes
  • Consider showing fewer lines on mobile to reduce scrolling
  • Line numbers may make mobile code blocks harder to read - use judiciously

<script>
function copyCode(button) {
    const codeBlock = button.closest('.code-block');
    const code = codeBlock.querySelector('code').textContent;
    navigator.clipboard.writeText(code);
    button.innerHTML = '<i data-lucide="check"></i> Copied';
    setTimeout(() => {
        button.innerHTML = '<i data-lucide="copy"></i> Copy';
        lucide.createIcons();
    }, 2000);
}
</script>
import { useState } from 'react';

const CodeBlock = ({ code, language = 'javascript', title, showLineNumbers }) => {
  const [copied, setCopied] = useState(false);

  const handleCopy = async () => {
    await navigator.clipboard.writeText(code);
    setCopied(true);
    setTimeout(() => setCopied(false), 2000);
  };

  return (
    <div className="code-block">
      {title && (
        <div className="code-block-header">
          <span className="code-block-title">{title}</span>
          <button
            className="btn btn-ghost btn-sm code-block-copy"
            onClick={handleCopy}
          >
            <i data-lucide={copied ? 'check' : 'copy'} />
            {copied ? 'Copied' : 'Copy'}
          </button>
        </div>
      )}
      <pre className={showLineNumbers ? 'line-numbers' : ''}>
        <code className={`language-${language}`}>{code}</code>
      </pre>
    </div>
  );
};

// Usage
<CodeBlock
  code="console.log('Hello, World!');"
  language="javascript"
  title="example.js"
/>
<template>
  <div class="code-block">
    <div v-if="title" class="code-block-header">
      <span class="code-block-title">{{ title }}</span>
      <button
        class="btn btn-ghost btn-sm code-block-copy"
        @click="handleCopy"
      >
        <i :data-lucide="copied ? 'check' : 'copy'" />
        {{ copied ? 'Copied' : 'Copy' }}
      </button>
    </div>
    <pre :class="{ 'line-numbers': showLineNumbers }">
      <code :class="`language-${language}`">{{ code }}</code>
    </pre>
  </div>
</template>

<script>
export default {
  props: {
    code: { type: String, required: true },
    language: { type: String, default: 'javascript' },
    title: String,
    showLineNumbers: Boolean
  },
  data() {
    return {
      copied: false
    };
  },
  methods: {
    async handleCopy() {
      await navigator.clipboard.writeText(this.code);
      this.copied = true;
      setTimeout(() => {
        this.copied = false;
      }, 2000);
    }
  }
}
</script>
