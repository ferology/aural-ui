Progress

Visual indicators for ongoing operations, loading states, and task completion. Show users how far along a process is or that work is happening in the background.

Basic Progress Bar
Uploading files... 65%

The basic progress bar shows determinate progress with a specific percentage. Always pair with a label describing the operation and the current percentage.

<div class="progress">
  <div class="progress-bar" style="width: 65%"></div>
</div>

Use semantic variants to convey status and meaning:

  • .progress (default) - Standard operations and neutral progress
  • .progress-success - Completed tasks or successful operations
  • .progress-warning - Approaching limits or requiring attention
  • .progress-error - Failed operations or critical states
  • .progress-info - Informational progress or downloads

Choose the appropriate size based on context:

  • .progress-sm - Inline progress in compact spaces or table rows
  • Default - Standard progress bars for most use cases
  • .progress-lg - More prominent progress for important operations
  • .progress-xl - Largest size, suitable for displaying percentage text inside the bar
Indeterminate Progress
Processing data...
Connecting to server...

Use indeterminate progress when you can't calculate the completion percentage. The animated bar indicates that work is happening without showing specific progress.

<div class="progress progress-indeterminate">
  <div class="progress-bar"></div>
</div>

When to use indeterminate:

  • Initial loading or connection establishment
  • Operations where duration is unknown
  • Processing steps without measurable progress
  • Waiting for external services or APIs

Best Practices

When to Use Progress Bars

  • File Uploads/Downloads: Show transfer progress with size and time estimates
  • Form Completion: Indicate how many steps remain in multi-step forms
  • Data Processing: Display progress for long-running operations
  • Installation/Updates: Show software installation or update progress
  • Loading Content: Indicate initial page load or data fetch progress

Determinate vs Indeterminate

  • Use Determinate when you can calculate percentage (file uploads, multi-step forms, batch processing with known item count)
  • Use Indeterminate when duration is unknown (connecting to services, processing with unknown length, waiting for external responses)
  • Transition: Start with indeterminate, then switch to determinate when percentage becomes available

Accessibility

  • Always include a text label describing the operation
  • Show percentage for determinate progress (visually or in label)
  • Use role="progressbar" and aria-valuenow, aria-valuemin, aria-valuemax attributes
  • Provide time estimates when possible ("2 minutes remaining")
  • Don't rely solely on color to convey meaning - combine with text and icons
  • Ensure sufficient color contrast for progress bars (WCAG AA minimum)

Design Guidelines

  • Always show context: Label what's progressing and current status
  • Provide feedback: Update progress frequently enough to feel responsive (every 100-500ms)
  • Show completion: Don't hide progress bars immediately - let users see 100% briefly
  • Allow cancellation: For long operations, provide a way to cancel
  • Handle errors: Switch to error variant if operation fails, with clear messaging
  • Time estimates: Show remaining time for operations over 10 seconds

Common Patterns

Loading State with Progress:

  <div class="card-body">

      <span>Loading dashboard data...</span>
      <span style="float: right;">45%</span>
    </div>
    <div class="progress">
      <div class="progress-bar" style="width: 45%"></div>
    </div>
  </div>
</div>

Upload with Cancel:

<div style="display: flex; gap: 12px; align-items: center;">

      <div class="progress-bar" style="width: 42%"></div>
    </div>
  </div>

</div>
