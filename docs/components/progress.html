When to Use Progress Bars

  • File Uploads/Downloads: Show transfer progress with size and time estimates
  • Form Completion: Indicate how many steps remain in multi-step forms
  • Data Processing: Display progress for long-running operations
  • Installation/Updates: Show software installation or update progress
  • Loading Content: Indicate initial page load or data fetch progress
  • Resource Usage: Display storage, memory, or quota consumption

Determinate vs Indeterminate

Use Determinate

When you can calculate percentage completion:

  • File uploads and downloads with known size
  • Multi-step forms with defined number of steps
  • Batch processing with known item count
  • Resource usage with known limits

Use Indeterminate

When duration or progress is unknown:

  • Connecting to services or establishing connections
  • Processing with unknown length or complexity
  • Waiting for external responses or API calls
  • Initial loading when total size is unknown

Transitioning States

Start with indeterminate, then switch to determinate when percentage becomes available. This provides immediate feedback while calculating progress.

Accessibility

  • Always include a text label describing the operation
  • Show percentage for determinate progress (visually or in label)
  • Use role="progressbar" with aria-valuenow, aria-valuemin, and aria-valuemax attributes
  • Provide time estimates when possible ("2 minutes remaining")
  • Don't rely solely on color to convey meaning - combine with text and icons
  • Ensure sufficient color contrast for progress bars (WCAG AA minimum)
  • Update progress frequently enough to feel responsive (every 100-500ms)

Best Practices

Size Selection

  • Small: Inline progress in compact spaces or table rows
  • Default: Standard progress bars for most use cases
  • Large: More prominent progress for important operations
  • Extra Large: Largest size, suitable for displaying percentage text inside the bar

Color Usage

  • Default: Standard operations and neutral progress
  • Success: Completed tasks or successful operations
  • Warning: Approaching limits or requiring attention
  • Error: Failed operations or critical states
  • Info: Informational progress or downloads

Design Guidelines

  • Always show context: Label what's progressing and current status
  • Provide feedback: Update progress frequently enough to feel responsive
  • Show completion: Don't hide progress bars immediately - let users see 100% briefly
  • Allow cancellation: For long operations, provide a way to cancel
  • Handle errors: Switch to error variant if operation fails, with clear messaging
  • Time estimates: Show remaining time for operations over 10 seconds

Mobile Considerations

  • Progress bars automatically scale for mobile devices
  • Consider using larger sizes on mobile for better visibility
  • Keep progress labels concise on small screens
  • Ensure cancel buttons meet minimum touch target size (44x44px)

const Progress = ({ value = 0, max = 100, variant, size, showLabel }) => {
  const percentage = Math.min(100, Math.max(0, (value / max) * 100));

  const classNames = [
    'progress-bar',
    variant && `progress-bar-${variant}`,
    size && `progress-bar-${size}`
  ].filter(Boolean).join(' ');

  return (
    <div
      className="progress"
      role="progressbar"
      aria-valuenow={value}
      aria-valuemin="0"
      aria-valuemax={max}
    >
      <div className={classNames} style={{ width: `${percentage}%` }}>
        {showLabel && <span className="progress-label">{Math.round(percentage)}%</span>}
      </div>
    </div>
  );
};

// Usage
<Progress value={65} showLabel />
<Progress value={80} variant="success" />
<Progress value={45} variant="warning" size="lg" showLabel />
<template>
  <div
    class="progress"
    role="progressbar"
    :aria-valuenow="value"
    aria-valuemin="0"
    :aria-valuemax="max"
  >
    <div :class="barClasses" :style="{ width: `${percentage}%` }">
      <span v-if="showLabel" class="progress-label">
        {{ Math.round(percentage) }}%
      </span>
    </div>
  </div>
</template>

<script>
export default {
  props: {
    value: { type: Number, default: 0 },
    max: { type: Number, default: 100 },
    variant: String,
    size: String,
    showLabel: Boolean
  },
  computed: {
    percentage() {
      return Math.min(100, Math.max(0, (this.value / this.max) * 100));
    },
    barClasses() {
      return [
        'progress-bar',
        this.variant && `progress-bar-${this.variant}`,
        this.size && `progress-bar-${this.size}`
      ].filter(Boolean).join(' ');
    }
  }
}
</script>

<!-- Usage -->
<Progress :value="65" show-label />
<Progress :value="80" variant="success" />
<script>
  export let value = 0;
  export let max = 100;
  export let variant = '';
  export let size = '';
  export let showLabel = false;

  $: percentage = Math.min(100, Math.max(0, (value / max) * 100));
  $: barClasses = [
    'progress-bar',
    variant && `progress-bar-${variant}`,
    size && `progress-bar-${size}`
  ].filter(Boolean).join(' ');
</script>

<div
  class="progress"
  role="progressbar"
  aria-valuenow={value}
  aria-valuemin="0"
  aria-valuemax={max}
>
  <div class={barClasses} style="width: {percentage}%">
    {#if showLabel}
      <span class="progress-label">{Math.round(percentage)}%</span>
    {/if}
  </div>
</div>
