Inputs

Text input fields with validation and various states.

Text input fields are fundamental form elements that allow users to enter and edit text. They come with built-in validation states and flexible styling options to provide clear feedback and maintain consistency across your application.

API Reference

Class Description Type
.input Base input class for text fields Required
.input-sm Small input variant Modifier
.input-lg Large input variant Modifier
.error Error state styling Modifier
.success Success state styling Modifier
.form-error Error message text Optional
.form-success Success message text Optional
disabled Disabled state attribute Modifier

Framework Examples

<input type="text" class="input" placeholder="Enter text...">
<input type="email" class="input error" placeholder="invalid-email">
<p class="form-error">Please enter a valid email address</p>
const Input = ({ error, errorMessage, ...props }) => {
  return (
    <div>
      <input
        className={`input ${error ? 'error' : ''}`}
        {...props}
      />
      {error && <p className="form-error">{errorMessage}</p>}
    </div>
  );
};

// Usage
<Input
  type="email"
  placeholder="Enter email"
  error={true}
  errorMessage="Please enter a valid email address"
/>
<template>
  <div>
    <input
      :class="['input', { error }]"
      v-bind="$attrs"
    />
    <p v-if="error && errorMessage" class="form-error">
      {{ errorMessage }}
    </p>
  </div>
</template>

<script>
export default {
  props: {
    error: Boolean,
    errorMessage: String
  }
}
</script>

Add icons to inputs using the .input-group wrapper with .input-group-icon elements. Use these classes to position icons:

  • .input-group-prefix - Adds padding for left icon
  • .input-group-suffix - Adds padding for right icon
  • .input-group-icon-left - Position icon on the left
  • .input-group-icon-right - Position icon on the right

Available sizes: .input-group-sm, default, and .input-group-lg

Textarea with Character Counter
Characters: 0 / 200
View Code 
<div class="form-group">
    <label class="label">Bio</label>
    <div class="input-with-counter">
        <textarea class="input" id="bio-input" maxlength="200" rows="4"
                  placeholder="Tell us about yourself..."
                  oninput="updateCounter()"></textarea>
        <div class="input-counter" id="bio-counter">
            <span class="input-counter-text">Characters:</span>
            <span class="input-counter-count">0 / 200</span>
        </div>
    </div>
</div>

<script>
function updateCounter() {
    const input = document.getElementById('bio-input');
    const counter = document.getElementById('bio-counter');
    const count = input.value.length;
    const max = input.maxLength;

    counter.querySelector('.input-counter-count').textContent = `${count} / ${max}`;

    counter.classList.remove('input-counter-warning', 'input-counter-error');
    if (count >= max * 0.8 && count < max) {
        counter.classList.add('input-counter-warning');
    } else if (count >= max) {
        counter.classList.add('input-counter-error');
    }
}
</script>

Character counters help users stay within input limits. The counter automatically updates as users type:

  • Default style when under 80% of limit
  • .input-counter-warning - Orange when 80-99% of limit
  • .input-counter-error - Red when at or over limit

Use the .input-with-counter wrapper to enable character counting.

Best Practices

When to Use

  • Use text inputs for single-line text entry such as names, emails, and search queries
  • Use textarea inputs for multi-line text entry such as comments, descriptions, and messages
  • Provide clear labels that describe what information is expected
  • Use placeholder text to provide examples or formatting hints, but never as a replacement for labels

Validation and Feedback

  • Show validation states (error/success) after the user has interacted with the field or attempted to submit
  • Provide clear, actionable error messages that explain what went wrong and how to fix it
  • Use success states sparingly - only when confirmation is valuable to the user
  • Display helper text for additional context or requirements before the user interacts with the field

Accessibility

  • Always associate labels with inputs using the for attribute or by wrapping the input
  • Ensure error messages are announced to screen readers using aria-describedby
  • Provide sufficient color contrast between text and backgrounds
  • Make sure inputs are keyboard accessible and have visible focus states
  • Use appropriate input types (email, tel, url) to improve mobile experience

Icons and Decorations

  • Use prefix icons to indicate the type of input (search, email, lock for password)
  • Use suffix icons for actions related to the input (show/hide password, clear field)
  • Keep icons simple and ensure they're semantically meaningful
  • Maintain consistent icon sizing across different input sizes

Character Limits

  • Display character counters for inputs with length constraints
  • Show warning state when approaching the limit (80-99%)
  • Show error state when at or over the limit
  • Consider whether a hard limit (preventing input) or soft limit (showing warning) is appropriate

Sizing

  • Use default size for most use cases
  • Use small inputs in compact layouts or secondary forms
  • Use large inputs to increase prominence in primary actions or improve touch targets
  • Maintain consistent sizing within a form or related group of inputs
