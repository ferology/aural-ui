Always Open (Multiple)
This panel stays open when you open another panel. Try clicking the other sections - this one will remain expanded!
Multiple panels can be open at the same time. This is useful when users need to compare content across sections.
Click to expand this section. Notice how the other sections stay open - they won't collapse automatically.
Great for documentation, FAQs, or any content where users might want to view multiple sections at once.

Add .accordion-always-open to allow multiple panels to be expanded simultaneously. By default, accordions close other panels when a new one opens.

<script>
document.querySelectorAll('.accordion-header').forEach(header => {
    header.addEventListener('click', () => {
        const expanded = header.getAttribute('aria-expanded') === 'true';
        const panel = document.getElementById(header.getAttribute('aria-controls'));

        header.setAttribute('aria-expanded', !expanded);
        panel.hidden = expanded;
    });
});
</script>
const Accordion = ({ items }) => {
  const [openItems, setOpenItems] = useState([]);

  const toggleItem = (id) => {
    setOpenItems(prev =>
      prev.includes(id)
        ? prev.filter(item => item !== id)
        : [...prev, id]
    );
  };

  return (
    <div className="accordion">
      {items.map(item => (
        <div key={item.id} className="accordion-item">
          <button
            className="accordion-header"
            aria-expanded={openItems.includes(item.id)}
            onClick={() => toggleItem(item.id)}
          >
            <span>{item.title}</span>
            <svg className="accordion-icon" viewBox="0 0 16 16" fill="currentColor" width="20" height="20">
              <path d="M4.22 6.22a.75.75 0 011.06 0L8 8.94l2.72-2.72a.75.75 0 111.06 1.06l-3.25 3.25a.75.75 0 01-1.06 0L4.22 7.28a.75.75 0 010-1.06z"/>
            </svg>
          </button>
          {openItems.includes(item.id) && (
            <div className="accordion-panel" role="region">
              <div className="accordion-content">
                {item.content}
              </div>
            </div>
          )}
        </div>
      ))}
    </div>
  );
};
<template>
  <div class="accordion">
    <div
      v-for="item in items"
      :key="item.id"
      class="accordion-item"
    >
      <button
        class="accordion-header"
        :aria-expanded="openItems.includes(item.id)"
        @click="toggleItem(item.id)"
      >
        <span>{{ item.title }}</span>
        <svg class="accordion-icon" viewBox="0 0 16 16" fill="currentColor" width="20" height="20">
          <path d="M4.22 6.22a.75.75 0 011.06 0L8 8.94l2.72-2.72a.75.75 0 111.06 1.06l-3.25 3.25a.75.75 0 01-1.06 0L4.22 7.28a.75.75 0 010-1.06z"/>
        </svg>
      </button>
      <div v-if="openItems.includes(item.id)" class="accordion-panel" role="region">
        <div class="accordion-content">
          {{ item.content }}
        </div>
      </div>
    </div>
  </div>
</template>

<script>
export default {
  props: {
    items: { type: Array, required: true }
  },
  data() {
    return {
      openItems: []
    };
  },
  methods: {
    toggleItem(id) {
      const index = this.openItems.indexOf(id);
      if (index > -1) {
        this.openItems.splice(index, 1);
      } else {
        this.openItems.push(id);
      }
    }
  }
}
</script>

Best Practices

When to Use

  • Use accordions to organize content into collapsible sections
  • Great for FAQs, settings panels, and navigation menus
  • Useful when you want to reduce visual clutter while keeping content accessible
  • Best for content that doesn't need to be visible all at once

Accessibility

  • Always use aria-expanded to indicate the open/closed state
  • Link headers to panels with aria-controls
  • Use role="region" on panels for proper semantics
  • Ensure keyboard navigation works (Enter/Space to toggle)
  • The accordion icon should rotate to indicate state changes

Content Guidelines

  • Keep header text concise and descriptive
  • Don't nest accordions more than one level deep
  • Consider showing the first item expanded by default
  • Use icons sparingly and only when they add meaning

Variants

  • .accordion - Default accordion with subtle styling (only one panel open at a time)
  • .accordion-bordered - Adds visible borders between items
  • .accordion-flush - Removes outer borders for embedded use
  • .accordion-always-open - Allows multiple panels to be expanded simultaneously
