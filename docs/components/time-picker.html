Framework Examples

<input type="time" class="time-picker" value="14:30">

<script>
const picker = document.querySelector('.time-picker');
picker.addEventListener('change', (e) => {
  console.log('Time selected:', e.target.value);
});
</script>
const TimePicker = ({ value, onChange }) => {
  const [time, setTime] = useState(value || '12:00');

  const handleChange = (e) => {
    const newTime = e.target.value;
    setTime(newTime);
    onChange?.(newTime);
  };

  return (
    <input
      type="time"
      className="time-picker"
      value={time}
      onChange={handleChange}
    />
  );
};

// Usage
const [selectedTime, setSelectedTime] = useState('14:30');
<TimePicker value={selectedTime} onChange={setSelectedTime} />
<template>
  <input
    type="time"
    class="time-picker"
    v-model="time"
    @change="handleChange"
  />
</template>

<script>
export default {
  props: {
    modelValue: { type: String, default: '12:00' }
  },
  data() {
    return {
      time: this.modelValue
    }
  },
  methods: {
    handleChange() {
      this.$emit('update:modelValue', this.time);
    }
  }
}
</script>

Best Practices

When to Use Time Pickers

  • Appointment Scheduling: Perfect for scheduling meetings, appointments, or events where users need to select specific times.
  • Time-based Settings: Ideal for alarms, reminders, timers, or any feature that requires time configuration.
  • Booking Systems: Essential for restaurant reservations, service bookings, or any time-slot selection.
  • Form Time Input: When users need to input time data in a structured, validated format.
  • Time Range Selection: Use multiple time pickers for start/end times in scheduling scenarios.

When NOT to Use Time Pickers

  • Duration Entry: If you need duration (e.g., "2 hours 30 minutes"), use separate hour/minute inputs or a duration field instead.
  • Relative Time: For relative times like "in 5 minutes" or "2 hours ago", use preset buttons or relative time inputs.
  • Time Ranges with Specific Slots: If only specific time slots are available, use a dropdown or radio buttons showing available times.
  • Quick Time Entry: For power users who can type quickly, consider allowing direct keyboard input alongside the picker.

Design Guidelines

Format Selection

Choose between 12-hour and 24-hour format based on your user's locale and preferences. 12-hour format is common in the US, while 24-hour format is preferred in many other regions.

Minute Intervals

Use appropriate minute intervals based on use case. For precise timing, use 1-minute intervals. For appointments, 15 or 30-minute intervals are often sufficient and reduce scrolling.

Now Button

Include a "Now" button for quick selection of the current time. This is especially useful for logging events or creating records with current timestamps.

Clear Visual Feedback

Highlight the selected time clearly. Show the formatted time in the input field so users can verify their selection at a glance.

Keyboard Support

Allow users to type time directly into the input field. Parse common formats like "3pm", "15:30", "3:30 PM".

Accessibility Guidelines

  • ARIA Labels: Provide clear labels for all interactive elements. Use aria-label for icon buttons and proper labeling for time selections.
  • Keyboard Navigation: Support full keyboard navigation:
    • Tab to move between hour/minute/period selectors
    • Arrow keys (Up/Down) to scroll through options
    • Enter/Space to select time
    • Escape to close dropdown without applying changes
  • Screen Reader Support: Announce selected time and changes to screen reader users. Include time format in announcements (e.g., "2:30 PM" or "14:30").
  • Focus Management: When opening the dropdown, set focus to the current hour or the first hour if no time is selected.
  • Touch Targets: Ensure scrollable lists and buttons are large enough for touch interaction (minimum 44x44px).
  • Visual Clarity: Use sufficient color contrast for selected items and ensure the interface is usable by users with color blindness.

Common Patterns

Meeting Scheduler

Use two time pickers for start and end times. Validate that end time is after start time. Show meeting duration automatically.

Daily Alarm

Combine with a toggle switch to enable/disable. Display time prominently in large font. Include repeat options (daily, weekdays, weekends).

Booking Time Slots

Use minute intervals matching your booking increments (15 or 30 minutes). Disable unavailable time slots. Show available slots prominently.

Business Hours

Use 24-hour format for clarity. Include separate pickers for open and close times. Allow setting different hours for different days.

Reminder/Timer

Include preset buttons for common times (5 min, 15 min, 1 hour). Make "Now" option easily accessible.

Technical Implementation

Time Formatting

Store time in a consistent format (preferably 24-hour or ISO 8601). Display in user's preferred format. Handle timezone conversions appropriately.

Validation

Validate time inputs and provide clear error messages. For time ranges, ensure end time is after start time. Consider business logic (e.g., no bookings after closing time).

Default Values

Set sensible defaults based on context. For appointments, default to next available slot. For alarms, default to common wake-up times. For current time entries, use "Now".

Mobile Optimization

On mobile, consider using native time pickers when appropriate. Ensure scrollable lists have sufficient height and smooth scrolling. Use bottom sheets for better reachability.
