Fullscreen Modal

Fullscreen modals take up the entire viewport with no margins or border radius. Perfect for:

  • Immersive content experiences (photo viewers, document editors)
  • Mobile-optimized full-page forms
  • Focus modes that require maximum screen space
  • Media galleries and slideshows

Use the .modal-fullscreen class to enable this variant.

Scrollable Modal

Scrollable modals have fixed headers and footers with an independently scrolling body. Benefits:

  • Header and footer remain visible while scrolling content
  • Better user experience for long content
  • Call-to-action buttons always accessible
  • Works with all modal sizes

Use the .modal-scrollable class to enable this variant.

1. Introduction

This modal demonstrates the scrollable variant, where the header and footer remain fixed while the body content scrolls independently. This is particularly useful for long-form content like terms of service, privacy policies, or detailed documentation.

2. Content Organization

The scrollable modal maintains visual hierarchy and keeps important actions (like Accept/Decline buttons) always visible at the bottom. This improves user experience by:

  • Providing clear navigation context with a fixed header
  • Ensuring call-to-action buttons are always accessible
  • Allowing users to scroll through content comfortably
  • Supporting various content lengths without layout issues

3. Use Cases

Scrollable modals are ideal for:

  • Legal Documents: Terms of service, privacy policies, user agreements
  • Long Forms: Multi-section forms that require scrolling
  • Documentation: Help articles, guides, tutorials
  • Product Details: Detailed product information, specifications
  • Release Notes: Changelog entries, update information

4. Technical Implementation

The scrollable modal uses CSS flexbox with specific overflow properties to achieve the fixed header/footer behavior:

  • The modal container uses display: flex and flex-direction: column
  • Header and footer have fixed dimensions
  • Body section has flex: 1 and overflow-y: auto
  • Maximum height is set to 90vh for comfortable viewing

6. Conclusion

The scrollable modal pattern provides an elegant solution for presenting lengthy content within a modal dialog. By keeping the header and footer fixed, users always have context about what they're viewing and can easily take action without scrolling to the bottom.

This implementation works seamlessly with all modal sizes (.modal-sm, default, .modal-lg) and maintains accessibility standards.
