Mobile-Only Visibility

Responsive Navigation

This bottom nav is visible only on mobile devices (below 1024px).

On desktop, you would use a sidebar or top navbar instead.

Use .aural-bottom-nav--mobile-only to hide the bottom navigation on desktop screens (1024px and wider).

Best Practices

When to Use

  • Use bottom navigation for mobile-first applications with 3-5 top-level destinations
  • Use when destinations need to be accessible from anywhere in the app
  • Use for equal-weight destinations that users switch between frequently
  • Use when thumbs need easy access to primary navigation
  • Ideal for content-focused apps (social media, e-commerce, news)

When Not to Use

  • Don't use for more than 5 destinations (use a hamburger menu instead)
  • Don't use on desktop unless it's a mobile-responsive design
  • Don't use for sequential navigation (use breadcrumbs or stepper instead)
  • Don't hide primary actions in bottom navigation
  • Don't use for apps with deep hierarchical navigation

Navigation Items

  • Keep to 3-5 items for optimal usability
  • Use clear, recognizable icons that match their labels
  • Keep labels short (1-2 words maximum)
  • Mark the current page with aria-current="page"
  • Use .aural-bottom-nav__item--active for visual active state
  • Order items by frequency of use (most used in the middle)

Icons and Labels

  • Always pair icons with labels for better recognition
  • Use 24x24px icons for optimal touch targets
  • Choose universally understood icons (home, search, profile)
  • For icon-only variant, always include aria-label
  • Ensure icons have sufficient contrast (3:1 minimum)

Badges and Notifications

  • Use .aural-bottom-nav__badge to show notification counts
  • Add .aural-bottom-nav__badge--pulse for urgent notifications
  • Use .aural-bottom-nav__badge--dot for simple presence indicator
  • Keep badge counts under 99 (show "99+" for larger numbers)
  • Update badge counts in real-time when possible
  • Include badge information in ARIA labels (e.g., "Notifications, 3 unread")

Floating Action Button (FAB)

  • Use FAB for the single most important action in your app
  • Always include descriptive aria-label on FAB
  • Place FAB in the center of 4-5 nav items
  • Use .aural-bottom-nav--with-fab for proper spacing
  • FAB should trigger creation actions (new post, compose, add)

Accessibility

  • Include role="navigation" on the nav element
  • Add descriptive aria-label to the nav (e.g., "Main navigation")
  • Use aria-current="page" on the active navigation item
  • Ensure minimum 44x44px touch targets for all items
  • Support keyboard navigation with Tab and Enter keys
  • Provide clear focus indicators for keyboard users
  • Ensure color contrast meets WCAG AA standards (4.5:1)

Mobile Considerations

  • Position at bottom for easy thumb access on mobile devices
  • Use .aural-bottom-nav--mobile-only to hide on desktop
  • Add .has-bottom-nav class to body for proper page padding
  • Support safe area insets for devices with notches (iPhone X+)
  • Consider hiding on scroll for immersive content (videos, images)
  • Use .aural-bottom-nav--slide-out and .aural-bottom-nav--slide-in for animations

Performance

  • Use CSS transitions instead of JavaScript animations
  • Lazy load badge counts via API for better initial load
  • Avoid heavy JavaScript on navigation item clicks
  • Use will-change: transform sparingly for animations
  • Optimize icons (use icon fonts or inline SVG)

Color Variants

  • .aural-bottom-nav--primary - Bold primary color background
  • .aural-bottom-nav--dark - Dark background with backdrop blur
  • .aural-bottom-nav--blur - Frosted glass effect
  • Test color variants with your brand colors for consistency
  • Ensure text and icons remain legible on colored backgrounds

Layout Tips

  • Items are distributed evenly with flexbox
  • Max width per item is 120px to prevent overly wide buttons
  • Use .aural-bottom-nav--compact for more compact spacing
  • Add .aural-bottom-nav--divided for visual separators between items
  • Bottom nav height is approximately 72px (with safe area support)
