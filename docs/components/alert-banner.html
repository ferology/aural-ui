Alert Banner

Prominent banners for important messages and system-wide alerts.

Interactive Demo

Click a button to trigger an alert banner:

View Code 
// JavaScript to show alert
function showAlert(type, title, message) {
    const iconMap = {
        'success': 'check-circle',
        'error': 'alert-circle',
        'warning': 'alert-triangle',
        'info': 'info'
    };

    const alertHTML = `
    <div class="aural-alert-banner aural-alert-banner--${type}">
        <i data-lucide="${iconMap[type]}"></i>
        <div class="aural-alert-banner__content">
            <div class="aural-alert-banner__title">${title}</div>
            <div class="aural-alert-banner__message">${message}</div>
        </div>
        <button class="aural-alert-banner__close">
            <i data-lucide="x"></i>
        </button>
    </div>`;

    container.innerHTML = alertHTML;
}

Best Practices

When to Use

  • System-wide announcements and updates
  • Important status messages that persist
  • Maintenance or outage notifications
  • Promotional or feature announcements

Alert Types

  • .aural-alert-banner--success - Positive outcomes, confirmations
  • .aural-alert-banner--info - Informational messages, updates
  • .aural-alert-banner--warning - Caution, approaching limits
  • .aural-alert-banner--error - Critical issues, failures

Structure

  • Use .aural-alert-banner__title for the main heading (optional)
  • Use .aural-alert-banner__message for the description
  • Use .aural-alert-banner__close for dismiss button
  • Icons should be placed before the content

Accessibility

  • Use appropriate ARIA roles (role="alert" for critical messages)
  • Ensure close buttons have aria-label
  • Maintain sufficient color contrast
  • Don't rely solely on color to convey meaning

Best Practices

  • Keep messages concise and actionable
  • Provide a clear action when appropriate
  • Allow users to dismiss non-critical alerts
  • Position banners prominently at the top of the page
