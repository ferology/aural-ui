Best Practices

When to Use File Upload

  • Document Submission: Forms that require file attachments (resumes, applications, documents)
  • Media Upload: Photo galleries, avatar changes, media libraries
  • Bulk Operations: Importing data, uploading multiple files at once
  • Content Creation: Blog posts, articles, or any content that requires media attachments

Choosing the Right Variant

  • Drop Zone (default): Best for primary upload actions where drag-and-drop is expected
  • Button Style: Use for secondary upload actions or when space is limited
  • Image Grid: Perfect for photo galleries or when visual preview is important
  • Small Size: Use in compact forms or sidebars
  • Large Size: Use for prominent upload actions or empty states

File Validation

  • Always specify accepted file types using the accept attribute
  • Validate file size on the client side before upload to provide immediate feedback
  • Display clear error messages when files don't meet requirements
  • Show file constraints (accepted types, max size) near the upload zone
  • Validate files on the server side as well for security

Progress Indication

  • Always show upload progress for files larger than 1MB
  • Display percentage or size uploaded for clarity
  • Use appropriate status indicators (uploading, success, error)
  • Allow users to cancel uploads in progress
  • Show clear success confirmation when uploads complete

Accessibility Guidelines

  • Ensure the hidden file input is accessible via keyboard navigation
  • Provide clear labels and instructions for screen readers
  • Use aria-label on icon-only buttons
  • Announce upload progress and status changes to screen readers
  • Don't rely solely on color to convey status - use icons and text
  • Ensure sufficient color contrast for all text and icons
  • Make drag-and-drop optional - always provide click-to-browse alternative

User Experience

  • Show file previews, especially for images
  • Allow users to remove files before uploading
  • Display file names, sizes, and types clearly
  • Provide visual feedback for drag-and-drop (highlight drop zone)
  • Group multiple uploads together for bulk operations
  • Show total progress for multiple file uploads
  • Retain uploaded files if form validation fails

Security Considerations

  • Always validate file types and sizes on the server
  • Scan uploaded files for malware
  • Don't trust client-side MIME types - verify on server
  • Store uploaded files outside the web root when possible
  • Generate unique filenames to prevent overwrites
  • Implement rate limiting to prevent abuse

JavaScript Integration

Aural UI provides automatic file upload functionality. To customize behavior, you can initialize with options:

// The component automatically handles:
// - Drag and drop
// - File selection
// - Preview generation
// - Progress tracking
// - File removal

// Custom validation example:
const input = document.querySelector('.file-upload__input');
input.addEventListener('change', (e) => {
    const files = Array.from(e.target.files);
    const maxSize = 10 * 1024 * 1024; // 10MB

    files.forEach(file => {
        if (file.size > maxSize) {
            alert(\`File "\${file.name}" is too large\`);
        }
    });
});
