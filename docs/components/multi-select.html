<script>
const select = document.querySelector('.form-select');
select.addEventListener('change', (e) => {
  const selected = Array.from(e.target.selectedOptions).map(opt => opt.value);
  console.log('Selected:', selected);
});
</script>
const MultiSelect = ({ options, value, onChange, placeholder, ...props }) => {
  const [isOpen, setIsOpen] = useState(false);
  const [selected, setSelected] = useState(value || []);

  const toggleOption = (optionValue) => {
    const newSelected = selected.includes(optionValue)
      ? selected.filter(v => v !== optionValue)
      : [...selected, optionValue];
    setSelected(newSelected);
    onChange?.(newSelected);
  };

  return (
    <div className="multi-select">
      <button onClick={() => setIsOpen(!isOpen)}>
        {selected.length ? `${selected.length} selected` : placeholder}
      </button>
      {isOpen && (
        <div className="multi-select-dropdown">
          {options.map(opt => (
            <label key={opt.value}>
              <input
                type="checkbox"
                checked={selected.includes(opt.value)}
                onChange={() => toggleOption(opt.value)}
              />
              {opt.label}
            </label>
          ))}
        </div>
      )}
    </div>
  );
};

// Usage
const [selected, setSelected] = useState([]);
<MultiSelect
  options={options}
  value={selected}
  onChange={setSelected}
  placeholder="Select options"
/>
<template>
  <div class="multi-select">
    <button @click="isOpen = !isOpen">
      {{ selectedText }}
    </button>
    <div v-if="isOpen" class="multi-select-dropdown">
      <label v-for="option in options" :key="option.value">
        <input
          type="checkbox"
          :value="option.value"
          v-model="selected"
        />
        {{ option.label }}
      </label>
    </div>
  </div>
</template>

<script>
export default {
  props: {
    options: Array,
    modelValue: Array,
    placeholder: { type: String, default: 'Select options' }
  },
  data() {
    return {
      isOpen: false,
      selected: this.modelValue || []
    }
  },
  computed: {
    selectedText() {
      return this.selected.length
        ? `${this.selected.length} selected`
        : this.placeholder;
    }
  }
}
</script>

Best Practices

When to Use Multi-Select

  • Multiple Related Choices: When users need to select 2 or more items from a predefined list (e.g., skills, tags, categories)
  • Filtering Operations: For applying multiple filters to data tables or search results
  • Permission Assignment: When assigning multiple roles, permissions, or access levels
  • Tag Management: For categorizing content, products, or resources with multiple labels
  • Team/Group Selection: When assigning tasks or sharing items with multiple people
  • Moderate Option Count: Best for 3-20 options. With search functionality, can handle more

When NOT to Use Multi-Select

  • Single Selection: Use a regular dropdown or radio buttons when only one option is allowed
  • Too Many Options (>20): Use a combobox or autocomplete with search instead. Large lists become overwhelming in dropdowns
  • Very Few Options (2-3): Consider checkboxes displayed inline instead for better visibility
  • Binary Choices: Use a toggle or single checkbox for yes/no or on/off selections
  • Complex Hierarchies: Use a tree select or nested menu for deeply nested categories
  • Long Text Options: If options have lengthy descriptions, consider alternative patterns like a list with checkboxes

Accessibility

  • Keyboard Navigation: Support Tab, Enter, Space, Arrow keys for navigation and selection
  • ARIA Labels: Use role="combobox", aria-multiselectable="true", aria-expanded
  • Screen Reader Support: Announce selection count and changes (e.g., "3 items selected")
  • Focus Management: Move focus logically between tags, dropdown, and options
  • Clear Labels: Always provide visible labels describing what's being selected
  • Remove Action: Ensure tag removal buttons are keyboard accessible and clearly announced
  • Visual Indicators: Use clear visual states for hover, focus, and selected items

Common Patterns

  • Skills/Technologies: Job applications, developer profiles, project requirements (with max limit)
  • Tags/Categories: Blog posts, product categorization, content management (unlimited or high limit)
  • Filters: E-commerce filtering, data table filtering, search refinement (with Select All/Clear All)
  • Permissions: User role management, access control, feature flags (with checkboxes for clarity)
  • Team Assignment: Task assignment, document sharing, project collaboration (with search for large teams)
  • Time Ranges: Selecting multiple days of the week, date ranges, scheduling (grouped options)
  • Multi-Language: Content translation, locale selection, internationalization settings

Implementation Tips

  • Show Selection Count: Display "X items selected" when tags are hidden or collapsed
  • Limit Visual Clutter: Consider showing only first few tags with "+X more" for many selections
  • Search for Scale: Add search/filter when options exceed 10-15 items
  • Group Related Items: Use group labels to organize options into logical categories
  • Validation Feedback: Show clear messages for required fields or max selection limits
  • Tag Order: Consider if tag order matters (reorderable) or is just visual (alphabetical)
  • Close on Outside Click: Dropdown should close when clicking outside the component
  • Preserve Selections: Maintain selections when dropdown closes and reopens

JavaScript API

Initialize a multi-select component with the Aural.initMultiSelect() method:

const multiSelect = Aural.initMultiSelect('element-id', {
  searchable: true,        // Enable search functionality (default: true)
  selectAll: false,        // Add "Select All" option (default: false)
  maxSelections: null,     // Maximum number of selections (default: null)
  onChange: (values) => {} // Callback when selection changes
});

// API Methods
multiSelect.getSelected();      // Get array of selected values
multiSelect.select('value');    // Select a value programmatically
multiSelect.deselect('value');  // Deselect a value programmatically
multiSelect.clear();            // Clear all selections
