<script>
function toggleMobileMenu() {
    const navbar = document.querySelector('.aural-navbar');
    const isOpen = navbar.classList.toggle('aural-navbar--menu-open');
    const toggle = navbar.querySelector('.aural-navbar__toggle');
    toggle.setAttribute('aria-expanded', isOpen);
}
</script>

Resize your browser to mobile width (below 768px) to see the hamburger menu toggle.

const Navbar = ({ brand, links, onSignIn, onSignUp }) => {
  const [activeLink, setActiveLink] = useState(links[0]?.id);

  return (
    <nav className="aural-navbar" role="navigation" aria-label="Main navigation">
      <div className="aural-navbar__inner">
        <a href="#" className="aural-navbar__brand">
          <span>{brand}</span>
        </a>
        <div className="aural-navbar__nav">
          {links.map(link => (
            <a
              key={link.id}
              href={link.href}
              className={`aural-navbar__link ${activeLink === link.id ? 'aural-navbar__link--active' : ''}`}
              aria-current={activeLink === link.id ? 'page' : undefined}
              onClick={() => setActiveLink(link.id)}
            >
              {link.label}
            </a>
          ))}
        </div>
        <div className="aural-navbar__actions">
          <button className="btn btn-ghost btn-sm" onClick={onSignIn}>
            Sign In
          </button>
          <button className="btn btn-primary btn-sm" onClick={onSignUp}>
            Sign Up
          </button>
        </div>
      </div>
    </nav>
  );
};
<template>
  <nav class="aural-navbar" role="navigation" aria-label="Main navigation">
    <div class="aural-navbar__inner">
      <a href="#" class="aural-navbar__brand">
        <span>{{ brand }}</span>
      </a>
      <div class="aural-navbar__nav">
        <a
          v-for="link in links"
          :key="link.id"
          :href="link.href"
          :class="['aural-navbar__link', { 'aural-navbar__link--active': activeLink === link.id }]"
          :aria-current="activeLink === link.id ? 'page' : undefined"
          @click="activeLink = link.id"
        >
          {{ link.label }}
        </a>
      </div>
      <div class="aural-navbar__actions">
        <button class="btn btn-ghost btn-sm" @click="$emit('signIn')">
          Sign In
        </button>
        <button class="btn btn-primary btn-sm" @click="$emit('signUp')">
          Sign Up
        </button>
      </div>
    </div>
  </nav>
</template>

<script>
export default {
  props: {
    brand: { type: String, required: true },
    links: { type: Array, required: true }
  },
  data() {
    return {
      activeLink: this.links[0]?.id
    };
  },
  emits: ['signIn', 'signUp']
}
</script>

Best Practices

When to Use

  • Use navbar for primary site navigation accessible from every page
  • Keep navigation items between 4-7 for optimal scanability
  • Place the logo/brand on the left for better discoverability
  • Position primary CTAs (sign up, login) on the right side
  • Use sticky navbar for long-scrolling pages to maintain access to navigation

Logo and Branding

  • Always link the logo/brand to the homepage
  • Use .aural-navbar__brand for consistent logo styling
  • Keep logo height between 28-40px for optimal display
  • Include both icon and text for better brand recognition

Navigation Links

  • Use .aural-navbar__link--active with aria-current="page" for the current page
  • Keep link labels concise (1-2 words)
  • Order links by importance or logical flow
  • Group related links together
  • Use icons sparingly - only when they add clarity

Action Buttons

  • Use icon buttons (.aural-navbar__action) for secondary actions
  • Always include aria-label on icon-only buttons
  • Use badges (.aural-navbar__badge) to show notification counts
  • Add .aural-navbar__badge--pulse for important notifications
  • Limit action buttons to 3-4 to avoid clutter

Search Integration

  • Use .aural-search-bar component for integrated search
  • Keep search width around 200-300px for desktop
  • Include search icon for visual clarity
  • Use placeholder text like "Search..." or "Search products..."
  • Consider a search modal for mobile instead of inline search

Mobile Considerations

  • Use .aural-navbar__toggle for hamburger menu on mobile
  • Toggle .aural-navbar--menu-open class to show/hide mobile menu
  • Ensure toggle button is at least 44x44px for touch targets
  • Include aria-expanded attribute on toggle button
  • Consider hiding search bar on mobile or moving it to menu

Positioning Variants

  • .aural-navbar--sticky - Stays at top when scrolling
  • .aural-navbar--fixed - Always fixed at top (add .has-navbar-fixed to body)
  • .aural-navbar--transparent - For hero sections or image backgrounds
  • .aural-navbar--blur - Modern frosted glass effect

Accessibility

  • Always include role="navigation" on the nav element
  • Add descriptive aria-label to nav (e.g., "Main navigation")
  • Use aria-current="page" on the active link
  • Include aria-label on all icon-only buttons
  • Add aria-expanded on mobile menu toggle
  • Ensure sufficient color contrast for all text (4.5:1 minimum)
  • Support keyboard navigation with proper focus states

Performance

  • Use position: sticky instead of JavaScript scroll listeners
  • Optimize logo images for web (WebP format recommended)
  • Lazy load dropdown menus when needed
  • Avoid heavy animations on scroll for better performance
