Indeterminate State

Use indeterminate state for "select all" checkboxes when only some items are selected.

View Code 
const selectAll = document.getElementById('select-all');
const children = document.querySelectorAll('.child-checkbox');

selectAll.indeterminate = true; // Set indeterminate state
selectAll.addEventListener('change', () => {
    children.forEach(cb => cb.checked = selectAll.checked);
});

Best Practices

When to Use

  • Multiple selections - When users can select zero, one, or multiple options from a list
  • Binary settings - For toggleable features like "Remember me" or "Show preview"
  • Agreeing to terms - For accepting policies or terms of service
  • Filtering options - For multi-select filters in search or data tables

Organization

  • Use fieldsets - Wrap related checkboxes in <fieldset> with <legend>
  • Logical order - Arrange options in a logical sequence (alphabetical, by importance, etc.)
  • Group by category - Separate distinct groups with headings or visual dividers
  • Keep labels concise - Use short, clear labels; add descriptions only when needed

Accessibility

  • Label association - Always wrap inputs in <label> tags or use for attribute
  • Keyboard navigation - Ensure checkboxes are keyboard accessible (Space to toggle)
  • Screen reader support - Use proper ARIA labels for indeterminate states
  • Focus indicators - Maintain visible focus states for keyboard users

Common Patterns

  • Select all - Use indeterminate state for "select all" with partial selection
  • Settings panels - Group checkboxes by category with descriptive text
  • Terms acceptance - Use for required agreements with links to full terms
  • Filter panels - Allow multiple filter selections with clear apply/reset actions
