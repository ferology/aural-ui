<script>
let checkinDate = null;

const checkinPicker = Aural.initDatePicker('date-picker-checkin', {
  minDate: new Date(), // Today or later
  format: 'MM/DD/YYYY',
  onChange: (date) => {
    checkinDate = date;
    // Update checkout minimum date to be after check-in
    checkoutPicker.setMinDate(new Date(date.getTime() + 86400000)); // +1 day
    console.log('Check-in selected:', date);
  }
});

const checkoutPicker = Aural.initDatePicker('date-picker-checkout', {
  minDate: new Date(),
  format: 'MM/DD/YYYY',
  onChange: (date) => {
    console.log('Check-out selected:', date);
    console.log('Booking duration:', Math.ceil((date - checkinDate) / 86400000), 'days');
  }
});
</script>

Best Practices

When to Use

  • Booking & Reservations: Hotel bookings, flight reservations, appointment scheduling, and event registrations
  • Forms with Dates: Birth dates, employment dates, expiration dates, or any date input in forms
  • Date Selection: When users need to visually browse and select specific dates from a calendar
  • Date Range Requirements: When dates must fall within specific ranges (e.g., future only, within 90 days)
  • Visual Context Needed: When seeing day of week and month layout helps users choose dates
  • Recurring Schedules: Setting up recurring events, shift planning, or availability calendars

When NOT to Use

  • Known Dates: If users know exact dates (like their birthday), plain text input may be faster
  • Historical Dates: For very old dates (birth year 1920s), use separate dropdowns or text inputs
  • Approximate Dates: For "around when" questions, use month/year pickers or text inputs
  • Date Ranges Spanning Months: Use a dedicated date range picker component instead
  • Time Selection: Use time picker or datetime picker if time is also needed
  • Simple Year Selection: Use a dropdown for year-only selection

Accessibility

  • Keyboard Navigation: Support Tab, Enter, Arrow keys, Escape for full keyboard control
  • Focus Management: Move focus logically between input, calendar, and action buttons
  • Screen Reader Support: Announce current date, selected date, and disabled dates
  • ARIA Labels: Use proper ARIA attributes for calendar grid and navigation controls
  • Today Indication: Clearly mark today's date with border or special styling
  • Date Format: Display date format hint in input placeholder or helper text
  • Touch Targets: Ensure date cells are at least 44x44px for touch interfaces
  • Error Messages: Provide clear feedback when dates are invalid or outside allowed range

Keyboard Shortcuts

  • Tab: Move focus between input field, calendar, and buttons
  • Enter/Space: Open calendar dropdown, select focused date
  • Arrow Keys: Navigate between dates in calendar grid
  • Page Up/Down: Navigate to previous/next month
  • Home/End: Jump to first/last day of month
  • Escape: Close calendar dropdown without selecting
  • T: Jump to today's date

Date Format Patterns

Common Formats

  • MM/DD/YYYY - US Format (01/26/2026)
  • DD/MM/YYYY - European Format (26/01/2026)
  • YYYY-MM-DD - ISO 8601 Format (2026-01-26)
  • MMM DD, YYYY - Short Month (Jan 26, 2026)
  • MMMM DD, YYYY - Full Month (January 26, 2026)
  • DD MMM YYYY - Day First (26 Jan 2026)

Format Selection Guidelines

  • Use ISO format (YYYY-MM-DD) for technical/database contexts and international audiences
  • Use MM/DD/YYYY for US audiences familiar with this format
  • Use DD/MM/YYYY for European and most international audiences
  • Use Month DD, YYYY for user-facing displays requiring clarity (no ambiguity)
  • Always show format example in placeholder or helper text
  • Consider locale-aware formatting based on user's browser settings

Common Patterns

Booking Check-in/Check-out

Two date pickers where check-out must be after check-in. Update check-out minimum date dynamically when check-in changes. Calculate and display booking duration.

Birthday/Age Verification

Set maximum date to ensure minimum age requirements. Consider using separate month/day/year dropdowns for dates far in the past.

Event Scheduling

Disable weekends if weekday-only events. Highlight already-booked dates. Set minimum date to today or future.

Appointment Booking

Show available dates only, disable fully booked days. Provide "Next Available" button to jump to next open slot.

Expiration/Deadline Dates

Set minimum date to today. Provide quick select buttons for common durations (30 days, 90 days, 1 year).

Implementation Tips

  • Mobile Considerations: Use native date picker on mobile devices for better UX (type="date")
  • Today Button: Always include a "Today" button for quick access to current date
  • Clear Button: Provide easy way to clear selection, especially for optional fields
  • Visual Indicators: Mark today with border, selected date with background, disabled dates with opacity
  • Validation: Validate dates immediately and show clear error messages for invalid selections
  • Auto-close: Close calendar after date selection for single-date pickers
  • Input Fallback: Allow typing dates directly into input field with format validation
  • Localization: Support different start-of-week (Sunday vs Monday) based on locale
  • Month/Year Dropdowns: For selecting distant dates, add month/year dropdown navigation
  • Performance: Only render visible month to improve performance with complex disable logic

Date Restrictions

Common Restriction Patterns

  • Future Only: Appointments, bookings (minDate: today)
  • Past Only: Birth dates, historical events (maxDate: today)
  • Date Range: Limited booking windows, promotional periods
  • Weekdays Only: Business appointments, office visits
  • Weekends Only: Special events, weekend classes
  • Specific Days Disabled: Holidays, blackout dates, fully booked dates
  • Rolling Window: Next 90 days, within 6 months

JavaScript API

Initialize a date picker with the Aural.initDatePicker() method:

const datePicker = Aural.initDatePicker('element-id', {
  defaultDate: new Date(),        // Initial selected date (default: null)
  format: 'MM/DD/YYYY',           // Display format (default: 'MM/DD/YYYY')
  minDate: new Date(),            // Minimum selectable date (default: null)
  maxDate: null,                  // Maximum selectable date (default: null)
  disabledDates: (date) => false, // Function returning true for disabled dates
  disableWeekends: false,         // Disable Saturdays and Sundays (default: false)
  firstDayOfWeek: 0,             // 0 = Sunday, 1 = Monday (default: 0)
  showTodayButton: true,          // Show "Today" button (default: true)
  showClearButton: true,          // Show "Clear" button (default: true)
  closeOnSelect: true,            // Close calendar after selection (default: true)
  onChange: (date) => {},         // Callback when date changes
  onOpen: () => {},               // Callback when calendar opens
  onClose: () => {}               // Callback when calendar closes
});

// API Methods
datePicker.getDate();            // Get selected date as Date object
datePicker.setDate(new Date());  // Set date programmatically
datePicker.clear();              // Clear selected date
datePicker.open();               // Open calendar dropdown
datePicker.close();              // Close calendar dropdown
datePicker.setMinDate(date);     // Update minimum date
datePicker.setMaxDate(date);     // Update maximum date
datePicker.goToMonth(year, month); // Navigate to specific month
datePicker.destroy();            // Clean up event listeners

Validation & Error States

  • Invalid Format: Show error if manually typed date doesn't match format
  • Out of Range: Indicate when selected date is before min or after max
  • Disabled Date: Prevent selection and show tooltip explaining why
  • Required Field: Mark field as required and validate on form submission
  • Date Logic Errors: Validate end date is after start date in date ranges
