When to Use Empty States

  • No Data: When a user's list, dashboard, or feed has no content yet
  • No Search Results: When a search or filter returns zero results
  • Error States: When content fails to load or an error occurs
  • First-Time Experience: To guide new users through initial setup or onboarding
  • Empty Collections: Folders, albums, or categories with no items
  • Permissions Issues: When users lack access to view certain content

Accessibility

  • Use semantic HTML with proper heading hierarchy (h2, h3)
  • Ensure icons are decorative with aria-hidden="true" or provide meaningful alt text
  • Make action buttons keyboard accessible with clear focus states
  • Use descriptive button text that explains the action
  • Ensure sufficient color contrast for text and icons
  • Don't rely solely on color to convey meaning (use icons and text together)
  • Keep descriptions concise but informative for screen reader users

Best Practices

Content Guidelines

  • Be Clear: Use simple, direct language that explains the situation
  • Be Helpful: Provide guidance on what users can do next
  • Be Encouraging: Use positive, supportive tone rather than negative language
  • Be Brief: Keep titles short (3-6 words) and descriptions concise (1-2 sentences)
  • Be Specific: Explain why the state is empty and what actions are available

Visual Design

  • Icons: Use relevant, recognizable icons that match the context
  • Icon Size: 48px is standard, adjust based on importance and space
  • Color: Use neutral colors for default states, semantic colors for errors/warnings
  • Spacing: Provide adequate whitespace around elements for visual clarity
  • Alignment: Center-align empty states for focus and visual balance

Actions

  • Primary Action: Always provide a clear primary action when possible
  • Multiple Actions: Offer alternatives when appropriate (create vs. import)
  • No Action: For informational states (all caught up), actions may not be needed
  • Error Recovery: Provide retry options and alternative paths forward
  • Button Text: Use action-oriented verbs (Create, Upload, Browse, Try Again)

Context Matters

  • First Use: Welcome users and guide them to get started
  • User Cleared: Acknowledge the action with positive reinforcement
  • No Results: Suggest ways to modify search or filters
  • Errors: Explain the issue and provide recovery options
  • Permissions: Clearly communicate access limitations

Common Mistakes to Avoid

  • Using negative or blaming language ("You have no items")
  • Providing empty states without any guidance or actions
  • Using generic icons that don't match the context
  • Making descriptions too long or technical
  • Forgetting to handle empty states during development
  • Using the same empty state for different contexts
  • Not providing a way forward for users
