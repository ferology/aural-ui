Best Practices

When to Use

  • Design Tools: Applications requiring precise color selection for graphics, UI design, or digital art
  • Theme Customization: Allowing users to personalize interface colors, brand colors, or appearance settings
  • Data Visualization: Configuring chart colors, graph series, or dashboard color schemes
  • Content Creation: Text editors, presentation tools, or any content requiring color formatting
  • Brand Guidelines: Managing and selecting from approved brand color palettes
  • CSS/Style Editing: Applications where users need to input or modify color values

When NOT to Use

  • Limited Color Options: Use simple color swatches or radio buttons for 5-10 predefined colors
  • Binary Choices: Don't use for selecting between just two colors - use buttons or toggles instead
  • Theme Mode Only: For simple light/dark mode, use a toggle switch
  • Preset Only Selection: If users should only pick from specific colors, use a swatch grid without the picker
  • Mobile-First Simple UIs: Color pickers can be complex on mobile; consider alternatives for mobile-only apps

Accessibility

  • Keyboard Navigation: Support Tab, Arrow keys, Enter/Space for all interactive elements
  • Visual Feedback: Provide clear focus states and hover effects on all interactive areas
  • Color Contrast: Display color contrast ratios for accessibility compliance (WCAG AA/AAA)
  • Hex Input: Always provide a text input for precise color entry by keyboard
  • Screen Reader Support: Label all inputs and controls with descriptive ARIA labels
  • Color Blindness: Include color value text alongside visual swatches
  • Large Touch Targets: Ensure handles and interactive areas are at least 44x44px for touch interfaces

Color Format Support

HEX Format

  • Most common format for web development (#RRGGBB or #RGB)
  • Supports 3-digit shorthand (#F00) and 6-digit full format (#FF0000)
  • Optional 8-digit format for alpha channel (#RRGGBBAA)
  • Best for: CSS, HTML, design handoff

RGB/RGBA Format

  • Red, Green, Blue channels (0-255) plus optional Alpha (0-1 or 0-100)
  • rgb(255, 0, 84) or rgba(255, 0, 84, 0.8)
  • More intuitive for understanding color composition
  • Best for: Programmatic color manipulation, transparency effects

HSL/HSLA Format

  • Hue (0-360Â°), Saturation (0-100%), Lightness (0-100%) plus optional Alpha
  • hsl(340, 100%, 50%) or hsla(340, 100%, 50%, 0.8)
  • Most human-friendly for understanding and adjusting colors
  • Best for: Color scheme generation, theme variations

Common Patterns

Theme/Brand Customization

Allow users to customize interface colors with live preview. Provide presets for primary, secondary, accent, and semantic colors. Store selections in user preferences or local storage.

Design Tool Color Selection

Full-featured picker with all input modes, eyedropper tool support, and recent colors. Enable copy/paste of color values and provide color history across sessions.

Document/Content Formatting

Compact picker in a popover triggered by a color swatch button. Focus on quick selection from presets with optional custom color access.

Data Visualization Configuration

Multiple color pickers for series, categories, or data ranges. Provide color scheme presets and ensure sufficient contrast for accessibility.

Brand Color Library Management

Display brand colors as presets with names/labels. Allow selection from approved colors while optionally permitting custom colors for special cases.

Implementation Tips

  • Default to HEX: Start with HEX mode as it's most familiar to web developers and designers
  • Validate Input: Validate hex codes and RGB/HSL ranges, showing clear error feedback
  • Auto-Update: Synchronize all representations when any input changes (hex, RGB, HSL, visual picker)
  • Preset Organization: Group presets logically (brand colors, grays, common colors) for easy scanning
  • Recent Colors: Save recently used colors per user session to speed up workflow
  • Copy on Click: Allow clicking the hex value to copy to clipboard for convenience
  • Eyedropper API: Use browser EyeDropper API when available for picking colors from screen
  • Alpha Channel: Make alpha/opacity control optional and hide when not needed
  • Responsive Design: Adjust picker size and layout for mobile screens (vertical layout, larger touch targets)
  • Color Contrast: Show WCAG contrast ratio when selecting text/background color pairs

JavaScript API

Initialize a color picker with the Aural.initColorPicker() method:

const colorPicker = Aural.initColorPicker('element-id', {
  color: '#F00054',           // Initial color (hex, rgb, or hsl string)
  alpha: true,                // Enable alpha/opacity control (default: false)
  mode: 'hex',                // Initial input mode: 'hex', 'rgb', 'hsl' (default: 'hex')
  presets: ['#f00054', ...],  // Array of preset colors to display
  recentColors: true,         // Track and display recently used colors (default: false)
  compact: false,             // Use compact variant (default: false)
  onChange: (color) => {},    // Callback when color changes
  onModeChange: (mode) => {}  // Callback when input mode changes
});

// API Methods
colorPicker.getColor();          // Get current color as hex string
colorPicker.setColor('#FF0000'); // Set color programmatically
colorPicker.getColorRGB();       // Get color as RGB object {r, g, b, a}
colorPicker.getColorHSL();       // Get color as HSL object {h, s, l, a}
colorPicker.destroy();           // Clean up event listeners
