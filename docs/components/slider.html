const Slider = ({ min = 0, max = 100, value, onChange, label, step = 1 }) => {
  return (
    <div className="aural-slider">
      <label className="aural-slider__label">
        <span>{label}</span>
        <span className="aural-slider__value">{value}</span>
      </label>
      <input
        type="range"
        className="aural-slider__input"
        min={min}
        max={max}
        value={value}
        step={step}
        onChange={(e) => onChange?.(Number(e.target.value))}
      />
    </div>
  );
};

// Usage
const [volume, setVolume] = useState(50);
<Slider label="Volume" value={volume} onChange={setVolume} />
<template>
  <div class="aural-slider">
    <label class="aural-slider__label">
      <span>{{ label }}</span>
      <span class="aural-slider__value">{{ modelValue }}</span>
    </label>
    <input
      type="range"
      class="aural-slider__input"
      :min="min"
      :max="max"
      :value="modelValue"
      :step="step"
      @input="$emit('update:modelValue', Number($event.target.value))"
    />
  </div>
</template>

  }
}
</script>

Best Practices

When to Use Sliders

  • Continuous Value Selection: Use sliders for selecting values from a continuous or large range where the exact value is less important than the relative position (e.g., volume, brightness, temperature).
  • Visual Feedback Matters: Ideal when users benefit from seeing the immediate effect of their adjustment, such as audio volume or image opacity.
  • Range with Context: When users need to understand their selection within the context of minimum and maximum values.
  • Approximate Values: Perfect for scenarios where approximate values are acceptable and the user wants quick, fluid adjustments.
  • Space-Efficient Controls: When you need a compact control that clearly shows the current value and available range.

When NOT to Use Sliders

  • Precise Input Required: Avoid sliders when users need to enter exact values. Use a number input field instead, especially for financial data, measurements, or quantities.
  • Binary Choices: Don't use sliders for simple on/off or yes/no decisions. Use a switch or toggle instead.
  • Small Value Ranges: For ranges with only a few options (2-5 discrete values), consider using radio buttons or a segmented control for clearer selection.
  • Non-Numeric Data: Sliders are inappropriate for categorical data or text-based options. Use dropdowns or radio buttons instead.
  • Critical Precision: Avoid in scenarios where precise control is difficult (e.g., selecting a specific date or entering a phone number).

Accessibility Guidelines

  • ARIA Labels: Always provide descriptive aria-label attributes that include the purpose, range, and current value.
  • Keyboard Navigation: Ensure sliders are fully keyboard accessible:
    • Arrow keys (Left/Right or Up/Down) for incremental adjustments
    • Home/End keys to jump to min/max values
    • Page Up/Page Down for larger increments
  • Value Display: Always show the current value visually, either inline or in a tooltip, so users know the selected value.
  • Orientation Attribute: For vertical sliders, include aria-orientation="vertical" to inform assistive technologies.
  • Touch Targets: Ensure the slider handle is large enough for touch interaction (minimum 44x44px as per WCAG guidelines).
  • Focus Indicators: Provide clear visual focus indicators for keyboard users.

Common Patterns

Volume/Audio Control

Use horizontal sliders with icons (speaker symbol) and percentage values. Consider muting at 0% and showing different speaker icons based on volume level.

Brightness/Contrast

Implement with min/max labels (Dark/Bright) and percentage display. Provide immediate visual feedback if controlling visible content.

Price Range Filtering

Use currency-formatted values with appropriate step increments (e.g., $50 or $100). Consider dual-handle range sliders for min/max price selection.

Zoom Level

Horizontal slider with percentage or ratio display (50%, 100%, 200%). Include reset-to-default functionality at 100%.

Rating/Quality Selection

Discrete steps (1-5 or 1-10) with appropriate labeling. Consider showing star icons or descriptive labels (Poor, Fair, Good, Excellent).

Time/Duration Selection

Format values as time units (minutes, hours). Use appropriate steps based on granularity needed (1 min, 5 min, 15 min).
