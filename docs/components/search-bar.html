Search Bar

Versatile search input components with autocomplete, filters, keyboard shortcuts, and command palette functionality for enhanced user experience.

Search bars are essential UI components for navigation and content discovery. They can be simple input fields or sophisticated command palettes with autocomplete, filters, and keyboard shortcuts. This component supports various sizes, styles, and functionality patterns to match your application's needs.

</script>

Best Practices

When to Use

  • Site-wide Search: Use search bars for navigation and content discovery across your application or website
  • Command Palette: Implement keyboard-driven interfaces for power users to quickly access features and commands
  • Filter Search: Enable users to narrow down large datasets or product catalogs with category filters
  • Autocomplete: Provide search suggestions to help users find content faster and reduce typing effort
  • Recent Searches: Display search history for frequently accessed content to improve user efficiency

Accessibility

  • Proper Labels: Always include aria-label on search inputs to describe their purpose for screen readers
  • Live Regions: Use aria-live="polite" on suggestion containers to announce results to screen reader users
  • Keyboard Navigation: Ensure full keyboard support with Tab, Enter, Escape, and Arrow keys for navigating suggestions
  • Focus Management: Trap focus within command palettes and return focus appropriately when closed
  • Clear Button: Provide a visible clear button that's accessible via keyboard with proper aria-label
  • Autocomplete Attributes: Use aria-autocomplete="list" and aria-expanded to indicate suggestion state
  • Role Assignment: Apply role="combobox" to the search wrapper and role="option" to suggestions

Common Patterns

  • Site Search: Traditional search bar in the header with autocomplete suggestions based on popular content
  • Command Palette: Modal-style search activated by keyboard shortcut (⌘K/Ctrl+K) for quick navigation and actions
  • Filter Search: Combine search with category chips or dropdown filters for refined results
  • E-commerce Search: Product search with filters, sorting, and image previews in suggestions
  • Documentation Search: Code-aware search with syntax highlighting in results and keyboard shortcuts
  • Multi-function Search: Support both navigation and actions (e.g., "Go to...", "Create...", "Toggle...")

Keyboard Shortcuts

  • ⌘K / Ctrl+K: Industry standard for opening command palette or focusing search
  • /: Alternative shortcut often used for quick search focus (common in forums and documentation)
  • Escape: Clear search input or close command palette/suggestions
  • ↑/↓ Arrow Keys: Navigate through search suggestions or command items
  • Enter: Select highlighted suggestion or execute search query
  • Tab: Move to next interactive element (clear button, filters, etc.)

Implementation Tips

  • Debounce search input to avoid excessive API calls (300-500ms recommended)
  • Show loading states while fetching suggestions to provide user feedback
  • Limit suggestion count to 5-8 items to prevent overwhelming users
  • Highlight matching text in suggestions to show why results were returned
  • Store recent searches in localStorage for persistence across sessions
  • Provide empty state messaging when no results are found
  • Consider mobile users with larger touch targets and appropriate keyboard behavior
